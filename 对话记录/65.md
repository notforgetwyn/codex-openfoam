## 阶段 28：CAD/几何导入 MVP（STL）

当前阶段目标：
- 同时推进三个方向：Python 内置可视化、工程化封装、CAD 导入。
- 先做可落地的 CAD/几何导入 MVP：支持 STL 导入到当前 OpenFOAM Case。
- 本阶段不做 STEP/IGES/CATIA/SolidWorks 的真实解析，因为这些需要 OCCT/CAD 内核、几何修复和三角化流程。

本阶段完成功能：
- 新增 `几何/CAD` 顶部菜单。
- 新增 `几何/CAD` 工作区标签页。
- 支持点击 `导入 STL`，选择本地 `.stl` 文件。
- STL 会复制到当前 Case 的 `constant/triSurface/` 目录。
- 导入后会写入 `constant/triSurface/geometry_manifest.json`，记录几何来源、Case 内路径、大小和导入时间。
- 支持刷新几何清单，在界面中查看当前 Case 已导入几何。
- 支持预览已导入 STL，使用 Python/VTK/Matplotlib 内置 3D 视图显示 STL 表面。
- 新增 STEP/IGES/CATIA/SolidWorks 支持说明，明确这些格式是后续 CAD 内核阶段。

本阶段页面与 UI 完成情况：

几何/CAD 页 `GeometryTab`：
- 页面标题：`几何/CAD`，使用 `QLabel` 渲染。
- UI 组件：使用 `QPushButton`、`QTextEdit`、`QHBoxLayout`、`QVBoxLayout`。
- 按钮或菜单项：`导入 STL` 可点击，打开文件选择框，只筛选 `.stl/.STL`。
- 按钮或菜单项：`刷新几何清单` 可点击，重新读取当前 Case 的 `geometry_manifest.json`。
- 按钮或菜单项：`预览已导入 STL` 可点击，选择已导入 STL 后打开独立 3D 视图。
- 按钮或菜单项：`STEP/IGES 支持说明` 可点击，弹窗说明后续需要 CAD 内核。
- 键盘操作：本阶段未新增快捷键。
- 鼠标操作：用户通过按钮完成导入、刷新和预览。
- 状态提示：导入完成后状态栏显示 `STL 几何导入完成`，预览完成后显示 `STL 预览已加载`。

顶部菜单：
- 新增 `几何/CAD` 菜单。
- 菜单项包括 `打开几何/CAD 页`、`导入 STL 几何`、`预览已导入 STL`。

3D 视图窗口 `VtkViewerDialog`：
- 新增 `plot_stl_file(path)`。
- 使用 VTK `vtkSTLReader` 读取 STL。
- 使用 Matplotlib `Poly3DCollection` 在独立 3D 窗口中显示 STL 表面。

本阶段完成设计：
- 新增 `GeometryImportService`，将几何导入逻辑从 UI 中拆出去。
- 服务层负责文件校验、复制、命名去重、manifest 写入和清单格式化。
- UI 层只负责选择文件、调用服务、刷新页面和触发 3D 预览。
- 当前只支持 STL，是因为 OpenFOAM 的 `constant/triSurface` 与 STL 流程最直接。
- STEP/IGES/CATIA/SolidWorks 暂不硬做，避免没有 CAD 内核时做出不可用的假功能。

本阶段数据流设计：
- 用户点击 `导入 STL` -> `QFileDialog` 返回 STL 路径 -> `GeometryImportService.import_stl()`。
- `import_stl()` -> 校验文件 -> 复制到 `当前 Case/constant/triSurface/` -> 写入 `geometry_manifest.json`。
- 用户点击 `刷新几何清单` -> `GeometryImportService.format_assets()` -> 页面显示导入记录。
- 用户点击 `预览已导入 STL` -> `GeometryImportService.list_assets()` -> 选择 STL -> `VtkViewerDialog.plot_stl_file()` -> 独立 3D 窗口显示。
- 本阶段新增 JSON：`constant/triSurface/geometry_manifest.json`。

- 工程化封装业务场景：新增几何导入服务和 manifest 记录。
- CAD 导入业务场景：完成 STL MVP；STEP/IGES/CATIA/SolidWorks 未完成。
