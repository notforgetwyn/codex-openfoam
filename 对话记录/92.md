## 阶段 74：一键仿真失败诊断与稳定网格修复

当前阶段目标：
- 修复一键仿真在 `icoFoam 最小求解` 阶段连续失败的问题。
- 让失败诊断能显示 OpenFOAM 原始 `FOAM FATAL ERROR` 段落，并识别数值发散、时间步过大这类退出码 8 问题。
- 修复绘制几何/计算域模板生成坏网格的问题，保证 `blockMesh -> checkMesh -> icoFoam` 能跑通。
- 本阶段不新增三维可视化功能、不新增新的求解器、不实现复杂弯管多块结构化网格。

本阶段完成功能：
- 失败诊断新增 `OpenFOAM 原始致命错误` 展示，用户能直接看到 OpenFOAM 报错的关键字、文件路径和函数来源。
- 诊断服务新增 `数值发散或时间步过大` 识别，能识别 `sigFpe`、超大残差和 Courant Number 超限。
- 修复 `fvSolution` 生成逻辑，参数保存后会自动包含 `pFinal`、`pRefCell`、`pRefValue`。
- 默认瞬态参数从 `deltaT=0.005/writeInterval=20` 调整为更稳的 `deltaT=0.001/writeInterval=100`。
- 修复计算域模板生成 `blockMeshDict` 的逻辑：实际 OpenFOAM 网格使用正体积外包盒，避免圆管/弯管单块 arc 网格产生负体积。
- 修复绘制几何页生成 `blockMeshDict` 的逻辑：对手绘计算域也先生成稳定外包盒网格。
- `求解器准备` 页只保留 `刷新求解器准备状态` 和 `补齐边界和物性` 两个准备按钮。

本阶段完成设计：
- `OpenFoamLogDiagnosticService` 负责解析 OpenFOAM 原始日志，UI 不直接解析 fatal 文本。
- `OpenFoamCaseParameterService` 负责生成完整 `fvSolution`，避免参数页保存时覆盖掉 `icoFoam` 必需配置。
- `ProjectService._block_mesh_dict_for_template()` 改为稳定外包盒网格生成策略。
- 绘制几何页仍可显示复杂形状预览，但实际 `blockMeshDict` 先使用稳定六面体背景域。
- 复杂圆管/弯管真实计算域后续应使用多块网格或 STL 管壁方案，不再用单块 arc 强行表达。

本阶段数据流设计：
- 用户点击一键仿真 -> 保存参数 -> 写入 `controlDict/fvSchemes/fvSolution/physicalProperties`。
- `case_parameter_service.save()` -> `_fv_solution_text()` -> 写入包含 `pFinal`、`pRefCell`、`pRefValue` 的 `system/fvSolution`。
- 计算域模板或绘制几何 -> 生成稳定外包盒 `blockMeshDict` -> `blockMesh` 生成正体积网格。
- `checkMesh` -> 若网格为 `Mesh OK` -> 进入 `icoFoam`。
- OpenFOAM 输出 -> `_current_process_output` -> `OpenFoamLogDiagnosticService.diagnose()` -> 问题面板中文诊断。
- 本阶段不新增 JSON 文件；仍使用已有 case 文件和参数配置文件。
