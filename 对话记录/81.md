## 阶段 47：3D 预览乱码修复与非盒状计算域说明

当前阶段目标：
- 修复 3D 预览图中 Matplotlib 中文文字在 WSL 环境下显示为方块/乱码的问题。
- 明确说明 OpenFOAM 中计算域不只能是正方体/长方体，后续可以扩展更多计算域形状。
- 本阶段只修复图内文字乱码，不新增新的非盒状计算域生成器。

本阶段完成功能：
- 将 Matplotlib 3D 图中的中文标题和提示改为英文/ASCII。
- 保留 Qt 主界面的中文 UI，不影响菜单、按钮、标签中文显示。
- 修复范围包括：
  - STL 放置预览标题。
  - 计算域与 STL 相对位置预览标题。
  - 3D 图中无数据、无压力点、无速度流线等提示。

本阶段页面与 UI 完成情况：

3D 预览图：
- 页面标题：由中文标题改为英文标题，避免 WSL 字体缺失导致乱码。
- UI 组件：`FigureCanvas`、Matplotlib 3D axes。
- 按钮或菜单项：未新增。
- 键盘操作：未新增。
- 鼠标操作：未新增。
- 状态提示：图内提示改为英文，Qt 日志和状态栏中文不变。

本阶段完成设计：
- 采用英文/ASCII 文案作为 Matplotlib 图内文字，避免依赖 Linux/WSL 中文字体配置。
- 不修改 Qt 字体系统，避免影响主界面中文显示。
- 后续如果要恢复图内中文，需要增加字体探测和 Matplotlib 字体注册逻辑。

本阶段数据流设计：
- 本阶段不改变业务数据流。
- 用户打开几何预览或 3D 视图 -> Matplotlib 绘制英文标题/提示 -> 避免乱码。
