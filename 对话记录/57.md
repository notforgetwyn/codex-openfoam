## 阶段 22：VTK StreamTracer 真实流线升级

当前阶段目标：
- 把速度流线从临时 NumPy 最近邻追踪升级为 VTK StreamTracer。
- 继续保持 Python 客户端内部可视化，不调用 ParaView。
- 本阶段不做用户自定义种子点密度、起点区域和流线长度参数，先把更可靠的 VTK 流线算法接入。

本阶段完成功能：
- `速度场 -> 加载速度流线` 现在使用 VTK StreamTracer 生成流线。
- 自动读取当前 Case 的真实 `U` 点字段。
- 自动判断主流向轴，例如 `X`。
- 自动在入口侧生成最多 24 个种子点。
- 使用 VTK `vtkRungeKutta4` 积分器追踪流线。
- 在独立 3D 视图窗口中绘制 VTK 输出的流线。
- 流线颜色继续表示速度大小 `|U|`。
- 结果文本区会显示算法、主流向轴、种子点数量、流线数量、流线点数和速度范围。

本阶段完成设计：
- 新增 VTK 依赖导入：`vtkPoints`、`vtkPolyData`、`vtkRungeKutta4`、`vtkStreamTracer`。
- `MainWindow._load_velocity_streamlines()` 改为调用 `_build_vtk_streamlines()`。
- 新增 `MainWindow._build_vtk_streamlines()`：负责种子点生成、StreamTracer 参数设置和 VTK 流线输出。
- `VtkViewerDialog.plot_velocity_streamlines()` 改为接收 VTK StreamTracer 输出的 polydata，并解析 `GetLines()` 绘制真实流线。
- 保持 UI 层和绘图层分离：MainWindow 负责数据和算法，VtkViewerDialog 负责显示。

本阶段数据流设计：
- 用户点击 `速度场 -> 加载速度流线`。
- `_load_velocity_streamlines()` 读取当前 Case 和当前时间步。
- `OpenFoamVtkService.build_geometry_filter()` 读取 OpenFOAM VTK 数据。
- 从 polydata 点字段中读取 `U`。
- `_build_vtk_streamlines()` 计算主流向和入口侧种子点。
- `vtkStreamTracer` 使用 `vtkRungeKutta4` 对 `U` 字段积分生成流线 polydata。
- `VtkViewerDialog.plot_velocity_streamlines()` 解析 VTK line cell 并绘制到 Matplotlib 3D 图。
- 结果区和日志区输出流线统计。
