## 阶段 42/43：自定义计算域与导入后 STL 位置编辑

当前阶段目标：
- 让用户可以手动输入计算域长宽高和网格数，不再只能使用固定模板。
- 让已经导入的 STL 可以重新设置缩放和平移，避免导入后发现位置不合适只能重新导入。
- 本阶段只做数值输入式编辑，不做鼠标拖拽、不做复杂 CAD 几何编辑、不做边界条件高级编辑。

本阶段完成功能：
- 几何/CAD 页新增自定义计算域输入：
  - `Lx/Ly/Lz`：计算域长宽高。
  - `Nx/Ny/Nz`：背景网格数量。
- 新增 `应用自定义计算域` 按钮。
- 应用自定义计算域后，会重写当前 Case 的 `system/blockMeshDict`。
- 应用自定义计算域后，会同步 `0/U` 和 `0/p` 的 `inlet/outlet/fixedWalls` 边界。
- 应用自定义计算域后，会自动计算建议 `locationInMesh`：
  - `X = Lx * 0.1`
  - `Y = Ly * 0.5`
  - `Z = Lz * 0.5`
- 新增 `编辑 STL 位置` 按钮。
- 用户可以选择当前 Case 已导入的 STL，并重新设置 scale、平移 X、平移 Y、平移 Z。
- STL 位置编辑会复用现有实时预览弹窗。
- STL 位置编辑完成后，会覆盖当前 Case 中 `constant/triSurface` 下的对应 STL，并更新 manifest 中的 transform。

本阶段页面与 UI 完成情况：

几何/CAD 页：
- 页面标题：`几何/CAD`。
- UI 组件：`QDoubleSpinBox`、`QSpinBox`、`QPushButton`、`QInputDialog`、`FigureCanvas`。
- 按钮或菜单项：
  - `应用自定义计算域`：根据用户输入的长宽高和网格数生成计算域。
  - `编辑 STL 位置`：选择已导入 STL 后打开缩放/平移编辑弹窗。
  - `刷新几何清单`：刷新 STL 清单和计算域预览。
- 键盘操作：本阶段未新增快捷键。
- 鼠标操作：用户调整自定义尺寸和网格后点击应用；用户选择 STL 后修改 scale/translate 并确认。
- 状态提示：日志会记录自定义计算域尺寸、网格数、建议 locationInMesh，以及 STL 新 transform。

STL 位置编辑弹窗：
- 页面标题：`导入 STL：位置和缩放`。
- UI 组件：缩放输入、X/Y/Z 平移输入、实时 3D 预览、OK/Cancel。
- 按钮或菜单项：
  - `OK`：确认并更新当前 Case 中的 STL。
  - `Cancel`：取消编辑，不改 Case 文件。
- 鼠标操作：调整数值后预览实时变化。
- 状态提示：如果 STL 不能读取，会在预览区显示错误说明。

本阶段完成设计：
- `ProjectService.apply_custom_domain()` 负责生成自定义计算域，并复用已有 `blockMeshDict` 生成逻辑。
- 自定义计算域仍采用统一边界规则：左侧 `inlet`，右侧 `outlet`，其余 `fixedWalls`。
- `GeometryImportService.update_stl_transform()` 负责更新已导入 STL 的坐标和 manifest。
- STL 重新变换优先使用原始 `source_path`，避免重复编辑时把变换叠加到已经变换过的 STL 上。
- 如果原始源文件不存在，则退回使用 Case 内 STL 作为基础文件。

本阶段数据流设计：
- 用户输入 Lx/Ly/Lz/Nx/Ny/Nz -> 点击应用自定义计算域 -> `ProjectService.apply_custom_domain()` -> 写入 `system/blockMeshDict` -> 同步 `0/U`、`0/p` -> 写入 `system/domain_config.json` -> UI 刷新预览。
- 用户点击编辑 STL 位置 -> 选择 STL -> 打开变换预览弹窗 -> 用户确认 -> `GeometryImportService.update_stl_transform()` -> 覆盖 `constant/triSurface/*.stl` -> 更新 `geometry_manifest.json` -> UI 刷新几何清单和预览。
