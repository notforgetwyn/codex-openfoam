## 阶段 26.1：3D 视图关闭清理与结果页布局修复

当前阶段目标：
- 修复关闭 3D 独立窗口后旧视图仍保留的问题。
- 修复结果页按钮过长导致主窗口被撑大、切回短页面后窗口不自动缩回的问题。
- 本阶段只做 UI 体验修复，不修改 OpenFOAM 求解、不修改可视化算法、不修改 Case 数据。

本阶段完成功能：
- 关闭 `FoamDesk 3D 视图` 独立窗口时，会自动清空所有 3D 标签页。
- 关闭 3D 窗口时，会释放当前 `Figure` 和 `FigureCanvas` 引用。
- 结果页顶部 8 个下拉按钮保持排成一行。
- 结果页按钮行外层新增横向滚动容器，按钮过宽时只在按钮区内部滚动，不再撑大主窗口。
- `清空视图`、`导出 PNG`、多标签页切换功能保持可用。

本阶段页面与 UI 完成情况：

3D 视图窗口 `VtkViewerDialog`：
- 页面标题：`FoamDesk 3D 视图`，未变更。
- UI 组件：继续使用 `QDialog`、`QTabWidget`、`FigureCanvas`。
- 按钮或菜单项：`清空视图` 保持可点击。
- 按钮或菜单项：`导出 PNG` 继续导出当前选中的标签页。
- 鼠标操作：点击窗口关闭按钮后，窗口内所有 3D 标签页都会清空。
- 状态提示：未新增弹窗提示，避免关闭窗口时打断用户。

结果页 `ResultsTab`：
- 页面标题：`结果`，未变更。
- UI 组件：新增 `QScrollArea` 包裹顶部按钮行。
- 按钮或菜单项：`结果数据 / 3D 视图 / 云图 / 速度场 / 字段时间 / 切面设置 / 流线设置 / 动画控制` 保持同一行。
- 鼠标操作：窗口宽度不够时，用户可以在按钮区域横向滚动查看后面的按钮。
- 状态提示：未变更。

本阶段完成设计：
- `VtkViewerDialog.closeEvent()` 调用 `_clear_tabs()`，保证关闭窗口就是清空当前 3D 会话。
- `_clear_tabs()` 不只删除 tab，也把 `figure` 和 `canvas` 设置为 `None`。
- 结果页将按钮行放入 `QScrollArea`，降低页面内容对主窗口最小宽度的影响。
- 保持按钮单行要求，同时避免单行按钮撑大整个主工作台。

本阶段数据流设计：
- 用户关闭 3D 窗口 -> `closeEvent()` -> `_clear_tabs()` -> 删除所有 tab -> 清空 figure/canvas 引用。
- 用户重新加载云图或速度场 -> `_reset_axes()` -> 创建新的 `FigureCanvas` -> 新增新的 3D 标签页。
- 用户进入结果页 -> 顶部按钮行进入 `QScrollArea` -> 按钮宽度超出时只影响滚动区，不影响主窗口尺寸。
- 本阶段不新增 JSON 读写，不修改 Case 文件。
