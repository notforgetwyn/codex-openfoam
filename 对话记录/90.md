## 阶段 72：仿真准备页与 Patch 边界条件表

当前阶段目标：
- 新增真正的 `仿真准备` 页面，承接完整仿真缺失项 5-6。
- 5-6 对应：`0/U`、`0/p`、`constant/physicalProperties`。
- 把 `0/U` 和 `0/p` 的 patch 边界条件做成可视化表格，让用户能按入口、出口、壁面、物体表面逐项配置。
- 保证后面的 `一键启动仿真` 按钮能够链接前面的求解器准备和仿真准备结果。

本阶段完成功能：
- 新增 `仿真准备` tab，位置在 `求解器准备` 和 `求解器选择` 之间。
- `仿真准备` 页面负责检查和补齐 `0/U`、`0/p`、`physicalProperties`。
- 新增 `Patch 边界条件表`，每一行对应一个 OpenFOAM patch。
- 表格列包括：`Patch`、`角色`、`U 类型`、`U 值`、`p 类型`、`p 值`。
- 自动从 `blockMeshDict` 读取边界名称。
- 如果当前 Case 有绘制几何导出的 STL，会自动把 `importedGeometry` 加入表格。
- 点击 `补齐边界和物性` 后，会按表格内容生成 `0/U`、`0/p` 和 `physicalProperties`。
- 一键启动仿真前会再次应用边界条件表，避免被默认字段生成逻辑覆盖。

本阶段页面与 UI 完成情况：

仿真准备页 `PhysicsPrepareTab`：
- 页面标题：`仿真准备`，使用 `QLabel` 渲染。
- UI 组件：`QPushButton`、`QTextEdit`、`QTableWidget`、`QVBoxLayout`、`QHBoxLayout`。
- 按钮或菜单项：
  - `刷新仿真准备状态`：检查 `0/U`、`0/p`、`constant/physicalProperties`。
  - `补齐边界和物性`：根据表格生成边界字段和物性文件。
  - `进入求解器选择`：跳转到求解器选择页。
  - `进入仿真参数`：跳转到仿真参数页。
  - `进入求解运行`：跳转到求解运行页。
  - `一键启动仿真`：调用统一流水线，并在启动前应用表格边界条件。
- 键盘操作：本阶段未新增快捷键。
- 鼠标操作：用户可以直接编辑表格中的角色、U 类型、U 值、p 类型、p 值。
- 状态提示：页面显示当前 Case、参数/物性来源、`0/U`、`0/p`、`physicalProperties` 的存在状态。

本阶段完成设计：
- 新增 tab 常量 `TAB_PHYSICS_PREPARE`，后续页面索引整体后移。
- 新增 `_build_physics_prepare_tab()` 构建仿真准备页。
- 新增 `_refresh_physics_prepare_panel()` 刷新 5-6 文件状态。
- 新增 `_prepare_physics_files()` 生成边界字段和物性文件。
- 新增 `_populate_boundary_table()` 从当前 Case 边界名称生成可编辑表格。
- 新增 `_apply_boundary_table_to_case()` 将表格内容写入 `0/U`、`0/p`、`physicalProperties`。
- 新增自定义 OpenFOAM 字段文本生成逻辑 `_custom_velocity_field()` 和 `_custom_pressure_field()`。

本阶段数据流设计：
- 绘制几何页 -> 生成 `blockMeshDict + STL`。
- 求解器准备页 -> 生成 `snappyHexMeshDict/controlDict/fvSchemes/fvSolution`。
- 仿真准备页 -> 读取 patch 名称 -> 展示边界条件表 -> 用户编辑表格。
- 用户点击 `补齐边界和物性` -> 表格数据 -> 写入 `0/U`、`0/p`、`constant/physicalProperties`。
- 一键启动仿真 -> 保存参数 -> 自动识别 STL -> 生成 snappy 配置 -> 应用边界条件表 -> 执行 OpenFOAM。
- 边界条件表会保存到 `system/foamdesk_boundary_table.json`，后续刷新页面时恢复。
