## 阶段 25：速度流线参数面板

当前阶段目标：
- 让用户能在结果页调整速度流线的生成参数，而不是只能使用固定的 VTK StreamTracer 默认配置。
- 本阶段只做流线参数配置和接入，不做新的流线算法、不做 ParaView 集成、不做复杂行业级后处理模板。

本阶段完成功能：
- 结果页新增 `流线设置` 下拉菜单。
- `流线设置` 内新增 `种子点` 参数，用来控制释放多少个流线起点。
- `流线设置` 内新增 `追踪长度` 参数，用来控制每条流线最多追踪多远。
- `流线设置` 内新增 `积分步长` 参数，用来控制流线追踪时每一步的计算粒度。
- 点击 `速度场 -> 加载速度流线` 时，会读取当前流线参数并传给 VTK StreamTracer。
- 速度流线加载后的日志和页面反馈会显示种子点上限、实际种子点数量、追踪长度、积分步长、流线数量和速度范围。

本阶段页面与 UI 完成情况：

结果页 `ResultsTab`：
- 页面标题：`结果`，继续使用 `QLabel` 渲染。
- UI 组件：继续使用 `QPushButton` + `QMenu` + `QWidgetAction` 组织下拉控制面板。
- 按钮或菜单项：
  - `结果数据`：保留刷新结果索引、导出求解指标、绘制残差曲线、导出 Markdown 报告。
  - `3D 视图`：保留加载 3D 技术验证场景、加载真实 OpenFOAM 3D Case。
  - `云图`：保留压力云图、压力表面云图、所选字段表面图。
  - `速度场`：保留速度箭头、速度切面、连续速度切面、速度流线。
  - `字段时间`：保留字段、时间步、刷新字段和加载所选字段表面图。
  - `切面设置`：保留切面方向和切面位置。
  - `流线设置`：新增种子点、追踪长度、积分步长。
  - `动画控制`：保留上一帧、下一帧、帧间隔、播放动画、暂停动画。
- 键盘操作：本阶段未新增快捷键。
- 鼠标操作：点击 `流线设置` 展开参数面板，修改参数后再点击 `速度场 -> 加载速度流线` 生效。
- 状态提示：加载流线时先显示“正在加载速度流线”，完成后显示“速度流线已加载”。

本阶段完成设计：
- 将 VTK StreamTracer 的关键硬编码参数提升为 UI 可配置参数。
- `MainWindow` 仍负责结果页交互编排，真实流线生成仍集中在 `_build_vtk_streamlines`。
- 参数边界在 UI 层和生成函数内都做了限制，避免用户输入导致 VTK 追踪参数异常。
- 本阶段保持 3D 视图独立窗口设计，避免 WSL 下 VTK/Qt 原生控件覆盖主界面。

本阶段数据流设计：
- 用户点击 `流线设置` -> 修改 `种子点 / 追踪长度 / 积分步长` -> 参数保存在当前 UI 控件状态中。
- 用户点击 `速度场 -> 加载速度流线` -> `_load_velocity_streamlines()` 读取 UI 参数 -> `openfoam_vtk_service` 读取当前 Case 的 VTK 几何和 `U` 字段 -> `_build_vtk_streamlines()` 使用参数生成 VTK 流线 -> `VtkViewerDialog.plot_velocity_streamlines()` 显示 3D 流线。
- 本阶段不新增 JSON 读写，流线参数暂时只保存在当前界面状态中。
- 本阶段不修改 Case 文件，不影响 OpenFOAM 求解结果。
