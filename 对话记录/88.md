## 阶段 70：绘制几何到完整仿真链路准备页

当前阶段目标：
- 在 `绘制几何` 页面之后新增承接页面，让用户生成 `blockMeshDict + STL` 后能继续完成完整仿真前缺失的准备工作。
- 打通流程顺序：绘制几何 -> 仿真准备 -> 求解器选择 -> 仿真参数 -> 求解运行。
- 本阶段不重写求解器算法、不新增新的 OpenFOAM 求解器，只把现有文件生成、参数保存和运行入口串起来。

本阶段完成功能：
- 新增 `仿真准备` 页面，位置在 `绘制几何` 和 `求解器选择` 之间。
- `仿真准备` 页面可以检查当前 Case 的 `blockMeshDict`、STL、`controlDict`、`fvSchemes`、`fvSolution`、物性文件、`0/U`、`0/p` 是否齐全。
- 新增 `补齐仿真文件` 按钮，会补齐求解器、参数、物性和边界字段文件。
- 新增 `进入求解器选择`、`进入仿真参数`、`进入求解运行` 按钮，形成明确流程入口。
- 新增 `一键启动仿真` 按钮，直接调用现有一键仿真流水线。
- 绘制几何导出的 STL 现在可以被几何服务自动识别，不再只依赖手动导入清单。
- 保留绘制几何草稿持久化：同一个 Case 下次打开会恢复上次绘制几何。

本阶段页面与 UI 完成情况：

仿真准备页 `SimulationPrepareTab`：
- 页面标题：`仿真准备`，使用 `QLabel` 渲染。
- UI 组件：`QPushButton`、`QTextEdit`、`QVBoxLayout`、`QHBoxLayout`。
- 按钮或菜单项：
  - `刷新准备状态`：重新检查当前 Case 的几何、参数和求解文件。
  - `补齐仿真文件`：生成或同步求解器支持文件、物性文件和边界字段。
  - `进入求解器选择`：跳转到求解器选择页并加载当前参数。
  - `进入仿真参数`：跳转到仿真参数页并加载当前参数。
  - `进入求解运行`：跳转到求解运行页并刷新运行摘要。
  - `一键启动仿真`：调用现有一键仿真流水线。
- 键盘操作：本阶段未新增快捷键。
- 鼠标操作：点击按钮完成检查、补齐、跳转或启动仿真。
- 状态提示：页面显示当前项目、Case、草稿、`blockMeshDict`、STL 数量、`snappyHexMeshDict`、参数和必需文件状态。

本阶段完成设计：
- 新增 tab 索引常量，避免新增页面后硬编码索引跳错页。
- 新增 `_build_simulation_prepare_tab()` 构建仿真准备页。
- 新增 `_refresh_simulation_prepare_panel()` 汇总当前 Case 是否具备完整仿真前置文件。
- 新增 `_prepare_simulation_from_geometry()` 负责补齐求解器、参数、物性和边界字段。
- 调整页面跳转为命名索引，适配新增页面后的 tab 顺序。
- `GeometryImportService.list_assets()` 增强为可自动识别 `constant/triSurface/*.stl`，支持绘制几何导出的 STL 进入后续 snappy/仿真链路。

本阶段数据流设计：
- 绘制几何页 -> `生成 blockMeshDict + STL` -> 写入 `system/blockMeshDict` 和 `constant/triSurface/*.stl`。
- 仿真准备页 -> `刷新准备状态` -> 检查当前 Case 文件系统 -> 显示缺失项和下一步建议。
- 仿真准备页 -> `补齐仿真文件` -> `ensure_minimal_case_template()` -> `case_parameter_service.save()` -> `ensure_solver_support_files()` -> 刷新求解运行摘要。
- 仿真准备页 -> `一键启动仿真` -> `_run_simulation_pipeline()` -> 保存参数 -> 生成 snappy 配置 -> `blockMesh -> snappyHexMesh -> checkMesh -> solver`。
- 绘制几何草稿 -> 写入 `当前 Case/.foamdesk/draw_geometry_state.json` -> 打开同一 Case 时自动恢复。
