# Sprint 9：结构化指标持久化与导出

当前阶段目标：
- 把 Sprint 8 提取出的求解指标保存为本地文件。
- 为后续 Python 内置曲线可视化提供稳定数据来源。
- 明确后续可视化不依赖 ParaView 外部软件，而是在 FoamDesk 内使用 Python 库完成。

本阶段完成功能：
- 新增求解指标导出服务。
- 支持导出 `metrics.json`。
- 支持导出 `residuals.csv`。
- 结果页新增“导出求解指标”按钮。
- 仿真成功完成后会自动导出求解指标。
- 导出目录为当前项目的 `results/`。
- `AGENTS.md` 新增可视化实现约束：
  - 第一版可视化不依赖 ParaView。
  - 优先使用 Python 库内嵌实现。
  - 曲线优先 `matplotlib` 或 `pyqtgraph`。
  - 后续字段/云图优先评估 `PyVista`、`VTK Python`、`meshio`。

本阶段页面与 UI 完成情况：

结果页 `MainWindow._build_results_tab()`：
- 页面标题：`结果`。
- UI 组件：`QPushButton`、`QTextEdit`。
- 按钮或菜单项：
  - `刷新结果索引`：扫描当前 case 运行产物。
  - `导出求解指标`：把当前运行日志指标导出为 JSON 和 CSV。
- 键盘操作：本阶段未新增快捷键。
- 鼠标操作：点击导出按钮后生成 `metrics.json` 和 `residuals.csv`。
- 状态提示：导出成功会写入日志和状态栏；没有运行日志时会提示先运行仿真。

本阶段完成设计：
- 新增 `MetricExportService`。
- `ApplicationContext` 注入指标导出服务。
- UI 层只负责触发导出和提示状态。
- 指标导出服务负责 JSON/CSV 文件格式。
- 数据格式为后续 Python 内置可视化服务复用，不绑定具体图表库。

本阶段数据流设计：
- OpenFOAM stdout/stderr -> `_current_process_output`。
- `_current_process_output` -> `OpenFoamLogMetricService.parse()` -> `SolverMetrics`。
- `SolverMetrics` -> `MetricExportService.export()`。
- 写入：
  - `project/results/metrics.json`
  - `project/results/residuals.csv`
