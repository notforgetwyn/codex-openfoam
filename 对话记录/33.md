# Sprint 12：OpenFOAM mesh/field 读取技术验证

当前阶段目标：
- 验证 FoamDesk 能否用 Python/VTK 读取真实 OpenFOAM case。
- 在 Sprint 11 的 VTK 三维视图基础上，从“单位盒子演示”推进到“真实 case reader”。
- 本阶段不做真实压力云图、速度矢量、切面和流线。

本阶段完成功能：
- 确认 `vtkOpenFOAMReader` 可用。
- 新增 OpenFOAM VTK 读取服务。
- 自动创建 VTK reader 需要的 `foamdesk.foam` marker 文件。
- 支持读取 OpenFOAM case 的时间步信息。
- 支持读取 VTK multiblock 输出 block 数量。
- 支持构建 VTK geometry filter，供 UI 渲染。
- 结果页新增 `加载真实 OpenFOAM 3D Case` 按钮。
- 点击按钮后尝试把当前项目 case 通过 VTK reader 加载到三维视图。

本阶段页面与 UI 完成情况：

结果页 `MainWindow._build_results_tab()`：
- 页面标题：`结果`。
- UI 组件：
  - `QPushButton`

  - `加载真实 OpenFOAM 3D Case`：通过 VTK OpenFOAM reader 读取当前 case。
- 键盘操作：本阶段未新增快捷键。

- 状态提示：加载真实 case 后输出 block 数量、时间步数量和 marker 文件路径。

本阶段完成设计：
- 新增 `OpenFoamVtkCaseInfo` 领域模型。
- 新增 `OpenFoamVtkService`。
- 服务层负责：
  - 创建 `foamdesk.foam` marker 文件。
  - 调用 `vtkOpenFOAMReader`。
  - 提取时间步和 block 信息。
  - 构建 geometry filter。
- UI 层只负责把 geometry filter 接入 VTK mapper/actor/renderer。
- 继续遵守“Python 内置可视化，不依赖 ParaView 外部软件”的规则。

本阶段数据流设计：
- 当前项目 `SimulationProject`。
- `OpenFoamVtkService.ensure_marker_file()` -> `case/foamdesk.foam`。
- `vtkOpenFOAMReader` 读取 marker 文件。
- reader 输出 multiblock 数据。
- `vtkCompositeDataGeometryFilter` 转为可渲染 geometry。
- UI 创建 mapper/actor 并显示到结果页 VTK 画布。
