# Sprint 8：残差与关键日志数据提取

当前阶段目标：
- 从 OpenFOAM 原始日志中提取第一批结构化求解指标。
- 在求解运行页展示关键指标摘要，为后续曲线可视化做准备。
- 本阶段不做图表，只做解析和文本摘要。

本阶段完成功能：
- 新增日志指标解析服务。
- 支持提取时间步 `Time = ...`。
- 支持提取残差：
  - 字段名，例如 `Ux`、`Uy`、`Uz`、`p`。
  - Initial residual。
  - Final residual。
  - No Iterations。
  - 所属时间步。
- 支持提取 Courant Number 的最大值。
- 支持提取 continuity errors 的 global 值。
- 求解运行页新增“关键指标摘要”区域。
- OpenFOAM 运行时，stdout/stderr 更新后会刷新关键指标摘要。

本阶段页面与 UI 完成情况：

求解运行页 `MainWindow._build_solver_run_tab()`：
- 页面标题：`求解运行`。
- UI 组件：新增 `关键指标摘要` 文本区域。
- 按钮或菜单项：运行、停止、刷新摘要保持原有逻辑。
- 键盘操作：本阶段未新增快捷键。
- 鼠标操作：用户点击运行后，指标摘要随日志逐步更新。
- 状态提示：显示时间步数量、最新时间、最新残差、最大 Courant Number、continuity global。

本阶段完成设计：
- 新增 `SolverResidual` 领域模型。
- 新增 `SolverMetrics` 领域模型。
- 新增 `OpenFoamLogMetricService`，专门负责日志指标解析。
- `ApplicationContext` 注入日志指标服务。
- `MainWindow` 在读取 OpenFOAM 输出时刷新指标摘要。
- 指标解析与 UI 解耦，后续可以直接复用这些数据做曲线图。

本阶段数据流设计：
- OpenFOAM stdout/stderr -> `_current_process_output`。
- `_current_process_output` -> `OpenFoamLogMetricService.parse()`。
- `SolverMetrics` -> `format_summary()`。
- 文本摘要 -> 求解运行页“关键指标摘要”。
