## 阶段 49：计算域模板扩展与前处理检查增强

当前阶段目标：
- 一次性扩展一组计算域模板，而不是一个模板拆一个 Sprint。
- 增强运行前检查，提前发现 Case 文件、OpenFOAM 环境、边界条件、STL 范围和 `locationInMesh` 问题。
- 本阶段优先保证 OpenFOAM `blockMesh` 可运行，不做复杂多 block 管网和真实 CAD 管路。

本阶段完成功能：
- 新增 `中等：圆管 6 x 2 x 2` 计算域。
- 新增 `高等：90 度弯管通道 4 x 4 x 1.2` 计算域。
- 圆管模板使用 `blockMesh` 的 `arc` 曲线边生成近似圆形截面。
- 90 度弯管模板使用 `blockMesh` 的 `arc` 曲线边生成弯曲通道。
- 计算域预览支持圆管线框。
- 计算域预览支持 90 度弯管线框。
- 几何/CAD 页新增 `运行前检查` 按钮。
- 运行前检查会检查：
  - `system/blockMeshDict`、`controlDict`、`fvSchemes`、`fvSolution`、`0/U`、`0/p` 是否存在。
  - OpenFOAM 环境脚本是否存在。
  - `blockMeshDict` 中的边界是否在 `0/U` 和 `0/p` 中都有配置。
  - `locationInMesh` 是否在计算域包围盒内。
  - 已导入 STL 的包围盒是否在计算域包围盒内。
  - snappy 最小加密等级是否小于等于最大加密等级。
  - 管道/弯管模板下导入障碍 STL 时给出提示。

本阶段页面与 UI 完成情况：

几何/CAD 页：
- 页面标题：`几何/CAD`。
- UI 组件：`QComboBox`、`QPushButton`、`FigureCanvas`、`QTextEdit`。
- 按钮或菜单项：
  - `计算域模板`：新增圆管和 90 度弯管通道。
  - `运行前检查`：检查当前 Case 在生成网格/运行仿真前的基础问题。
- 键盘操作：本阶段未新增快捷键。
- 鼠标操作：选择圆管/弯管模板后，预览图会显示对应线框；点击运行前检查后查看问题面板。
- 状态提示：检查通过会弹出提示；检查失败会显示错误并写入问题面板。

本阶段完成设计：
- `ProjectService.domain_vertices()` 继续作为计算域顶点统一入口。
- `ProjectService.domain_edge_lines()` 新增圆管和弯管的 `arc` 曲线边。
- 圆管和弯管仍用单 block，降低复杂度并保证 MVP 稳定。
- 运行前检查先使用包围盒检查，这是简单、快速、稳定的第一版。
- 对圆管/弯管这类非盒状计算域，当前检查仍是包围盒近似，后续可升级为精确几何内外判断。

本阶段数据流设计：
- 用户选择圆管/弯管模板 -> `ProjectService.domain_vertices()` + `domain_edge_lines()` -> 生成 `blockMeshDict` -> `blockMesh` 生成网格。
- 用户点击运行前检查 -> 读取 Case 必需文件 -> 读取 OpenFOAM 环境脚本 -> 读取边界条件 -> 读取 STL 顶点 -> 检查范围和参数 -> 写入问题面板。

本阶段核心业务场景完成进度：
- 计算域业务场景：完成圆管和 90 度弯管通道模板。
- 前处理业务场景：增加运行前检查，减少盲目运行失败。
- OpenFOAM 集成业务场景：圆管和弯管已通过 `blockMesh` 实测。
- STL 管理业务场景：能检查 STL 是否明显超出计算域。
- 未完成内容：多 block 圆管、真实曲线弯管、圆管内流专用边界条件、精确 STL 与非盒状计算域内外判断留到后续阶段。
