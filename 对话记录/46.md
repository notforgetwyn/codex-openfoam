## 阶段 15 修复：Case 级结果隔离与当前 Case 记忆

当前阶段目标：
- 修复切换到别的 case 后仍然看到旧 case 数据的问题。
- 让每个 case 拥有独立的求解指标和残差结果。
- 让项目记住当前选中的 case，避免重新打开项目后回到默认 case。

本阶段完成功能：
- `project.json` 新增/更新 `current_case`。
- 切换 case 后会把当前 case 写入 `project.json`。
- 新增 case 后也会把新 case 写入 `project.json`。
- 打开项目时优先读取 `current_case`，而不是永远打开默认 `case`。
- 求解指标 `metrics.json` 改为写入当前 case 的 `foamdesk_results` 目录。
- 残差文件 `residuals.csv` 改为写入当前 case 的 `foamdesk_results` 目录。
- 残差曲线读取当前 case 的 `foamdesk_results/residuals.csv`。
- 主工作台恢复已有结果时只读取当前 case 的结果，不再读取项目级旧结果。

本阶段页面与 UI 完成情况：

页面名 `MainWindow`：
- 页面标题：`FoamDesk`。
- UI 组件：顶部菜单栏、左侧 `Case 树`、工作区、底部日志面板。
- 按钮或菜单项：
  - 点击 `Case 树` 中的 case 会切换当前 case。
  - `运行` 只对当前 case 执行。
  - `结果` 页只显示当前 case 的结果索引。
- 鼠标操作：
  - 点击不同 case 后，参数页、求解页、结果页都切换到该 case。
- 状态提示：
  - 切换后显示 `Case 已切换。`
  - 已有结果读取当前 case 的 `foamdesk_results`。

本阶段完成设计：
- `ProjectService.open_project()` 优先读取 `project.json.current_case`。
- `ProjectService.switch_case()` 切换后写入当前 case。
- `ProjectService.create_case()` 创建后写入当前 case。
- `ProjectService.remember_project()` 同时保存最后项目路径和当前 case。
- `MetricExportService` 的输出目录从 `project/results` 改为 `project.case_dir/foamdesk_results`。
- `ResidualPlotService` 的读取目录从 `project/results` 改为 `project.case_dir/foamdesk_results`。
- `MainWindow._restore_project_result_state()` 改为读取当前 case 的结果目录。

本阶段数据流设计：
- 点击 Case 树中的 `132` -> `ProjectService.switch_case(project, "132")` -> 写入 `project.json.current_case = "132"`。
- 点击运行 -> OpenFOAM 在 `132` case 目录执行 -> 指标导出到 `132/foamdesk_results`。
- 打开结果页 -> 读取 `132` case 的时间步和字段文件。
- 绘制残差曲线 -> 读取 `132/foamdesk_results/residuals.csv`。
- 重新打开项目 -> `open_project()` 读取 `current_case` -> 自动恢复到 `132`。
