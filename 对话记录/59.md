## 阶段 24：连续速度切平面云图 v1

当前阶段目标：
- 把速度切面从点采样显示升级为连续插值云图。
- 本阶段先基于当前 OpenFOAM 点字段和 `scipy.griddata` 做二维规则网格插值。
- 不改变 OpenFOAM 求解流程，不做 VTK 原生切平面和任意角度切平面。

本阶段完成功能：
- `速度场` 下拉菜单新增 `加载连续速度切面`。
- 连续速度切面复用结果页的 `切面` 和 `位置` 控件。
- 支持 `自动/X/Y/Z` 切面方向。
- 支持 `0.00~1.00` 归一化切面位置。
- 自动读取当前 Case 的真实 `U` 点字段。
- 计算速度大小 `|U|`。
- 使用 `scipy.griddata` 将切面附近离散点插值为二维规则网格。
- 在独立 3D 视图中显示连续平面颜色云图。
- 右侧色条显示速度大小范围。
- 结果区显示请求方向、实际方向、切面位置、参与插值点数和速度范围。

本阶段完成设计：
- `VtkViewerDialog` 新增 `plot_velocity_slice_contour()`。
- `MainWindow` 新增 `_load_velocity_slice_contour()`。
- 连续切面与点采样切面共用切面参数，避免 UI 复杂化。
- 插值策略：先用 linear 插值，如果边界出现 NaN，再用 nearest 补齐。
- 当前实现属于 Python 侧后处理插值，后续可以升级为 VTK Cutter + Probe/Resample。

本阶段数据流设计：
- 用户选择 `切面=Y`、`位置=0.50`。
- 用户点击 `速度场 -> 加载连续速度切面`。
- `_load_velocity_slice_contour()` 读取当前 Case、时间步、切面方向、切面位置。
- `OpenFoamVtkService.build_geometry_filter()` 读取当前 OpenFOAM 结果。
- 从 VTK polydata 中读取点字段 `U`。
- `plot_velocity_slice_contour()` 计算速度大小 `|U|`。
- 根据方向和位置筛选切面附近点。
- 将切面点投影到二维平面坐标。
- `scipy.griddata()` 插值成规则二维网格。
- Matplotlib `plot_surface()` 绘制连续平面云图。
- 结果区和日志区输出统计信息。
