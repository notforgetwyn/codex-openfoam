## 阶段 23：Markdown 仿真报告导出 v1

当前阶段目标：
- 增加当前 Case 的 Markdown 报告导出能力。
- 本阶段先做稳定的文本报告，不做 PDF、不自动嵌入图片、不做复杂工程结论生成。
- 报告保存到当前 Case 的 `foamdesk_results/report.md`。

本阶段完成功能：
- 结果页 `结果数据` 下拉菜单新增 `导出 Markdown 报告`。
- 点击后会生成当前 Case 的 Markdown 报告。
- 报告内容包含项目名称、Case 名称、项目路径、Case 路径。
- 报告内容包含网格状态、最新时间步、时间步目录、字段文件列表。
- 报告内容包含 VTK marker、block 数量、可用时间步、点字段、单元字段。
- 报告内容包含求解指标摘要，例如时间步数量、Courant 数、continuity global、残差记录数量。
- 报告内容包含最新残差信息。
- 报告内容包含残差 CSV 路径。
- 报告内容列出当前 FoamDesk 已支持的可视化能力。
- 报告导出成功后，结果区显示报告路径和报告包含的内容说明。

本阶段完成设计：
- 新增 `ReportExportService`，专门负责报告内容生成和写入。
- `ApplicationContext` 新增 `report_export_service`。
- `MainWindow` 新增 `_export_markdown_report()`，只负责 UI 入口、调用服务、错误提示和结果反馈。
- 报告服务不依赖 UI，后续可以复用到批量报告、自动报告和命令行导出。
- 报告 v1 使用 Markdown，便于用户直接阅读，也便于后续转换 PDF。

本阶段数据流设计：
- 用户点击 `结果数据 -> 导出 Markdown 报告`。
- `_export_markdown_report()` 检查当前项目和 Case。
- `_export_markdown_report()` 调用 `ResultIndexService.index()` 获取结果索引。
- `_export_markdown_report()` 调用 `OpenFoamVtkService.inspect()` 获取 VTK 字段信息。
- `_export_markdown_report()` 调用 `ReportExportService.export_markdown()`。
- `ReportExportService` 读取 `foamdesk_results/metrics.json` 和 `residuals.csv` 状态。
- `ReportExportService` 生成 Markdown 文本。
- 报告写入 `当前Case/foamdesk_results/report.md`。
- UI 结果区显示导出路径。
