## 阶段 30：snappyHexMesh 执行入口

当前阶段目标：
- 在 Sprint 29 已能生成 `snappyHexMeshDict` 的基础上，增加点击按钮运行 `snappyHexMesh` 的入口。
- 本阶段只接入命令执行、实时日志、完成/失败状态，不做复杂参数面板、不做网格质量可视化、不做几何修复。

本阶段完成功能：
- 几何/CAD 页新增 `运行 snappyHexMesh` 按钮。
- 顶部 `几何/CAD` 菜单新增 `运行 snappyHexMesh` 菜单项。
- 点击运行时会检查 OpenFOAM 环境是否可用。
- 如果当前 Case 没有 `system/snappyHexMeshDict`，会先尝试根据已导入 STL 自动生成。
- 如果当前 Case 缺少 `system/blockMeshDict`，会阻止运行并提示原因。
- 执行命令链路为：`blockMesh && snappyHexMesh -overwrite`。
- 运行过程继续使用底部日志面板实时显示 stdout/stderr。
- 任务结束后能区分 `snappyHexMesh` 成功、失败和最小仿真成功、失败，不再都显示成“最小仿真”。

本阶段页面与 UI 完成情况：

几何/CAD 页 `GeometryTab`：
- 页面标题：`几何/CAD`，继续使用 `QLabel` 渲染。
- UI 组件：继续使用 `QPushButton`、`QTextEdit`、顶部 `QMenu` 和底部日志面板。
- 按钮或菜单项：
  - `导入 STL`：导入几何到 `constant/triSurface`。
  - `刷新几何清单`：刷新 STL 和 snappy 配置状态。
  - `生成 snappyHexMeshDict`：只生成配置文件，不运行命令。
  - `运行 snappyHexMesh`：执行 `blockMesh && snappyHexMesh -overwrite`。
  - `预览已导入 STL`：打开独立 3D 视图预览 STL。
  - `STEP/IGES 支持说明`：说明复杂 CAD 格式后续扩展路径。
- 键盘操作：本阶段未新增快捷键。
- 鼠标操作：点击 `运行 snappyHexMesh` 后切到底部日志面板，并显示实时 OpenFOAM 输出。
- 状态提示：
  - 运行中：`snappyHexMesh 运行中。`
  - 成功：`snappyHexMesh 完成。`
  - 失败：`snappyHexMesh失败，退出码 X。`

本阶段完成设计：
- `MainWindow` 新增 `_active_process_kind`，用于区分当前 `QProcess` 是最小仿真还是 snappyHexMesh。
- `MainWindow._run_snappy_hex_mesh()` 复用已有 `QProcess + bash -lc + source OpenFOAM bashrc` 执行模式。
- `MainWindow._on_process_finished()` 根据任务类型分别处理成功和失败状态。
- 本阶段没有新增独立命令执行服务，先复用现有流程；后续如果 OpenFOAM 命令变多，再抽象为统一 `OpenFoamCommandRunner`。

本阶段数据流设计：
- 用户点击 `运行 snappyHexMesh` -> 检查当前 Case -> 检查 OpenFOAM 环境 -> 检查/自动生成 `snappyHexMeshDict`。
- UI 创建 `QProcess` -> 执行 `source OpenFOAM bashrc && cd case && blockMesh && snappyHexMesh -overwrite`。
- OpenFOAM stdout/stderr -> `_read_process_stdout()` / `_read_process_stderr()` -> 底部日志面板。
- 进程结束 -> `_on_process_finished()` -> 根据 `_active_process_kind` 更新任务状态、结果索引和诊断面板。

本阶段核心业务场景完成进度：
- 主菜单业务场景：未变更。

- 几何/CAD 业务场景：完成 STL 导入、snappy 配置生成、snappy 网格命令执行入口。
- 求解业务场景：新增前处理网格生成命令入口，但还没有形成完整“几何 -> 网格 -> 求解 -> 结果”的自动流水线。
