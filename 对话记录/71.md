## 阶段 34：一键前处理流水线

当前阶段目标：
- 把“生成 snappyHexMeshDict -> 运行 snappyHexMesh -> 运行 checkMesh”串成一个按钮。
- 降低用户手动操作成本，避免每次都要按顺序点击多个入口。
- 本阶段只做串联执行，不做自动修复、不做复杂队列、不做网格参数推荐。

本阶段完成功能：
- 几何/CAD 页新增 `一键前处理` 按钮。
- 顶部 `几何/CAD` 菜单新增 `一键前处理` 菜单项。
- 点击后会先选择/确认当前 STL。
- 点击后会读取当前 snappy 参数面板，生成新的 `system/snappyHexMeshDict`。
- 生成配置后自动执行 `blockMesh && snappyHexMesh -overwrite && checkMesh`。
- 底部日志面板实时显示三步 OpenFOAM 命令输出。
- 流水线成功后，会把 `checkMesh` 的中文摘要写入问题/诊断面板。
- 流水线失败后，任务状态会显示 `一键前处理失败，退出码 X`。

本阶段页面与 UI 完成情况：

几何/CAD 页 `GeometryTab`：
- 页面标题：`几何/CAD`，继续使用 `QLabel` 渲染。
- UI 组件：继续使用 `QPushButton`、`QFormLayout`、`QTextEdit`、底部日志面板和问题/诊断面板。
- 按钮或菜单项：
  - `生成 snappyHexMeshDict`：只生成配置，不运行命令。
  - `运行 snappyHexMesh`：运行 `blockMesh && snappyHexMesh -overwrite`。
  - `运行 checkMesh`：只检查当前网格。
  - `一键前处理`：生成配置并串联执行 `blockMesh -> snappyHexMesh -> checkMesh`。
- 键盘操作：本阶段未新增快捷键。
- 鼠标操作：点击 `一键前处理` 后，如果有多个 STL，会弹出选择框；如果只有一个 STL，会直接使用该 STL。
- 状态提示：
  - 运行中：`一键前处理运行中。`
  - 成功：`一键前处理完成。`
  - 失败：`一键前处理失败，退出码 X。`

本阶段完成设计：
- `MainWindow._select_stl_asset_for_snappy()` 抽出 STL 选择逻辑，供流水线复用。
- `MainWindow._run_preprocess_pipeline()` 负责完整前处理编排。
- `_active_process_kind` 新增 `preprocess`，让结束回调区分一键前处理和单独命令。
- `_on_process_finished()` 对 `preprocess` 成功结果复用 `_format_check_mesh_summary()` 生成中文摘要。
- 本阶段仍复用单个 `QProcess`，暂不引入任务队列。

本阶段数据流设计：
- 用户点击 `一键前处理` -> 检查当前项目/Case -> 检查 OpenFOAM 环境。
- 选择 STL -> 读取 snappy 参数 UI -> 写入 `system/snappyHexMeshDict` 和 `snappy_config.json`。
- 启动 `QProcess` -> 执行 `source OpenFOAM bashrc && cd case && blockMesh && snappyHexMesh -overwrite && checkMesh`。
- stdout/stderr -> 底部日志面板。
- 进程结束 -> 根据 `_active_process_kind=preprocess` -> 刷新几何清单、结果索引、诊断摘要。

本阶段核心业务场景完成进度：
- 主菜单业务场景：未变更。
- 项目选择业务场景：未变更。
- Case 管理业务场景：一键前处理只作用于当前 Case。
- 几何/CAD 业务场景：完成从 STL、snappy 参数、网格生成到网格检查的前处理小闭环。
- 求解业务场景：前处理已能一键执行，但还没有自动进入 `icoFoam` 或其他求解器。
- 结果业务场景：成功后刷新结果索引，并输出 checkMesh 摘要。
- 未完成内容：前处理任务拆分进度条、失败步骤定位、自动修复建议、运行前参数校验说明、网格可视化检查留到后续阶段。
