# Sprint 7：结果文件管理与运行产物索引

当前阶段目标：
- 把“结果”页从占位文本改成真实结果索引页面。
- 先识别 OpenFOAM case 中已经生成的运行产物。
- 本阶段不做云图、不做曲线、不调用 ParaView/PyVista。

本阶段完成功能：
- 新增结果索引服务。
- 能识别 OpenFOAM 时间步目录，例如 `0`、`0.1`、`0.5`。
- 能识别最新时间步。
- 能识别 `constant/polyMesh` 网格目录是否存在。
- 能识别时间步目录中的字段文件：

  - `p`
  - `phi`
  - `T`
  - `k`
  - `epsilon`

  - `nut`
- “结果”页新增“刷新结果索引”按钮。
- 项目切换后自动刷新结果索引。
- 仿真正常完成后自动刷新结果索引。

本阶段页面与 UI 完成情况：

结果页 `MainWindow._build_results_tab()`：
- 页面标题：`结果`。
- UI 组件：`QLabel`、`QPushButton`、`QTextEdit`。
- 按钮或菜单项：
  - `刷新结果索引`：扫描当前 case 目录并展示结果。
- 键盘操作：本阶段未新增快捷键。
- 鼠标操作：点击刷新按钮更新结果索引。
- 状态提示：刷新成功后写入日志和状态栏；未选择项目时提示先打开项目。

本阶段完成设计：
- 新增 `CaseResultIndex` 领域模型。
- 新增 `ResultIndexService`，负责扫描 OpenFOAM 运行产物。
- `ApplicationContext` 注入结果索引服务。
- UI 层只负责展示索引结果，不直接遍历文件系统。
- 结果索引服务只做文件发现，不做数据解析和可视化，便于后续扩展。

本阶段数据流设计：
- 用户打开/选择项目 -> `ResultIndexService.index(project)` -> 生成 `CaseResultIndex` -> 结果页展示。
- 仿真完成 -> `_on_process_finished(exit_code=0)` -> 自动刷新结果索引。
- 用户点击刷新结果索引 -> 重新扫描 case 目录 -> 更新结果页。
