## 阶段 33：snappy 参数持久化回填 UI

当前阶段目标：
- 让用户生成过 `snappyHexMeshDict` 后，下次打开项目或切换 Case 时，几何/CAD 页能恢复上次使用的 snappy 参数。
- 本阶段只做参数读取和 UI 回填，不做参数版本管理、不做参数模板库、不做自动推荐。

本阶段完成功能：
- `GeometryImportService` 新增 `load_snappy_settings()`。
- 程序会读取当前 Case 的 `constant/triSurface/snappy_config.json`。
- 如果文件存在且格式正确，会还原 `SnappyHexMeshSettings`。
- 几何/CAD 页刷新时会自动把上次参数回填到 UI 控件。
- 打开项目、切换 Case、打开几何/CAD 页、生成 snappy 字典后刷新，都会触发参数回填。
- 如果没有 `snappy_config.json` 或文件损坏，程序不会报错，会继续使用 UI 默认值。

本阶段页面与 UI 完成情况：

几何/CAD 页 `GeometryTab`：
- 页面标题：`几何/CAD`，继续使用 `QLabel` 渲染。
- UI 组件：继续使用 `QFormLayout`、`QSpinBox`、`QDoubleSpinBox`、`QCheckBox`。
- 按钮或菜单项：
  - `刷新几何清单`：现在不仅刷新几何清单，也会尝试回填 snappy 参数。
  - `生成 snappyHexMeshDict`：生成后写入 `snappy_config.json`，随后刷新并回填当前参数。
  - `运行 snappyHexMesh`、`运行 checkMesh`：本阶段未改变命令逻辑。
- 键盘操作：本阶段未新增快捷键。
- 鼠标操作：用户切换项目/Case 后，打开几何/CAD 页可以看到上次保存的参数。
- 状态提示：本阶段未新增弹窗提示，避免每次刷新都打扰用户。

本阶段完成设计：
- `GeometryImportService._snappy_config_path()` 统一维护 `snappy_config.json` 路径。
- `GeometryImportService.load_snappy_settings()` 负责读取 JSON 并转换为 `SnappyHexMeshSettings`。
- `MainWindow._load_snappy_settings_into_form()` 负责把服务层返回的参数写入 UI 控件。
- `MainWindow._refresh_geometry_panel()` 作为统一触发点，保证项目打开、Case 切换、几何页打开和手动刷新都能复用同一套回填逻辑。
- 服务层对损坏 JSON 做容错，避免旧项目或手工修改文件导致 UI 崩溃。

本阶段数据流设计：
- 用户生成 snappy 字典 -> 服务层写入 `constant/triSurface/snappy_config.json`。
- 用户重新打开项目或切换 Case -> `_activate_project()` -> `_refresh_geometry_panel()`。
- `_refresh_geometry_panel()` -> `_load_snappy_settings_into_form()` -> `GeometryImportService.load_snappy_settings()`。
- 服务层读取 JSON -> 返回 `SnappyHexMeshSettings` -> UI 控件更新。
- 如果 JSON 不存在或读取失败 -> 返回 `None` -> UI 保持默认值。

本阶段核心业务场景完成进度：
- 主菜单业务场景：未变更。

- Case 管理业务场景：切换 Case 后会读取该 Case 自己的 snappy 参数，避免不同 Case 共用错误参数。
- 几何/CAD 业务场景：完成 snappy 参数编辑、保存到 JSON、从 JSON 回填 UI。
- 求解业务场景：未变更。
- 结果业务场景：未变更。
- 未完成内容：参数模板库、参数版本历史、参数重置默认、参数校验说明、从已有 `snappyHexMeshDict` 反向解析参数留到后续阶段。
