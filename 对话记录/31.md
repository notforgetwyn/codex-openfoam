# Sprint 10：第一版残差曲线可视化

当前阶段目标：
- 按用户要求使用 Python 库在 FoamDesk 内部完成第一版可视化。
- 不依赖 ParaView 外部软件。
- 先做 residual 曲线，不做云图、切面、流线和三维场显示。

本阶段完成功能：
- `pyproject.toml` 新增 `matplotlib` 依赖声明。
- 新增残差曲线数据服务。
- 支持读取 `results/residuals.csv`。
- 按字段分组残差数据，例如 `Ux`、`Uy`、`Uz`、`p`。
- 结果页新增 `绘制残差曲线` 按钮。
- 在 FoamDesk 内嵌 matplotlib 画布显示残差曲线。
- 仿真成功完成后：
  - 自动导出求解指标。
  - 自动尝试绘制残差曲线。

本阶段页面与 UI 完成情况：

结果页 `MainWindow._build_results_tab()`：
- 页面标题：`结果`。
- UI 组件：`QPushButton`、`QTextEdit`、`FigureCanvasQTAgg`。
- 按钮或菜单项：
  - `刷新结果索引`：刷新 OpenFOAM 运行产物索引。
  - `导出求解指标`：导出 `metrics.json` 和 `residuals.csv`。
  - `绘制残差曲线`：读取 `residuals.csv` 并绘制曲线。
- 键盘操作：本阶段未新增快捷键。
- 鼠标操作：点击 `绘制残差曲线` 后在结果页下方显示图。
- 状态提示：绘制成功后写入日志和状态栏；缺少 CSV 时提示先运行仿真或导出指标。

本阶段完成设计：
- 新增 `ResidualPlotService`，负责读取残差 CSV 并整理为字段序列。
- UI 层负责 matplotlib 画布和曲线绘制。
- 数据来源固定为 Sprint 9 生成的 `results/residuals.csv`。
- 第一版采用 log 纵轴显示 final residual，便于观察残差下降。
- 可视化路线确认：先曲线，再字段/网格/云图。

本阶段数据流设计：
- `results/residuals.csv` -> `ResidualPlotService.load_series()`。
- `dict[field, list[(time, residual)]]` -> matplotlib Figure。
- matplotlib Figure -> PySide6 `FigureCanvasQTAgg` -> 结果页展示。
- 仿真成功 -> 自动导出指标 -> 自动绘制残差曲线。

本阶段核心业务场景完成进度：
- Python 内置可视化业务场景：完成第一版 residual 曲线。
- 结果管理业务场景：结果页新增曲线画布。
- 指标导出业务场景：与曲线可视化联动。
- OpenFOAM 最小仿真业务场景：成功完成后可自动触发曲线。
- 未完成内容：Courant 曲线、continuity 曲线、字段云图、网格显示、交互式缩放工具栏留到后续阶段。
