## 阶段 16.1：可视化时间步切换接入 VTK Reader

当前阶段目标：
- 让结果页的时间步选择器真正参与数据读取。
- 用户选择某个时间步后，`加载所选字段表面图` 应按该时间步请求 VTK Reader。
- 本阶段不新增新的可视化图型，只完善已有字段表面图的数据来源。

本阶段完成功能：
- `OpenFoamVtkService.build_geometry_filter()` 支持 `time_value` 参数。
- 传入时间步时调用 VTK Reader 的 `SetTimeValue(time_value)`。
- 传入时间步时调用 `UpdateTimeStep(time_value)`。
- 结果页 `加载所选字段表面图` 会读取时间步下拉框当前值。
- 时间步文本会转换为 `float`。
- 所选时间步会传入 VTK Reader。
- 结果页提示文案改为：当前图像已按所选时间步请求 VTK Reader 读取。

本阶段页面与 UI 完成情况：

页面名 `ResultsTab`：
- 页面标题：`结果`。
- UI 组件：字段下拉框、时间步下拉框、`加载所选字段表面图` 按钮。
- 按钮或菜单项：
  - `刷新可视化字段`：刷新字段和时间步。
  - `加载所选字段表面图`：按所选字段和所选时间步绘图。
- 鼠标操作：
  - 用户选择 `p` 或 `U`。
  - 用户选择 `0.0`、`0.1`、`0.2` 等时间步。
  - 点击加载后绘制该字段在所选时间步下的表面图。
- 状态提示：
  - 结果页显示字段、时间步、面片数量和值范围。
  - 日志记录 `field`、`time`、`faces` 和 `range`。

本阶段完成设计：
- VTK 服务层负责时间步读取能力，不把 Reader 细节散落到 UI。
- UI 层只负责解析时间步下拉框并传给服务层。
- `_selected_visualization_time()` 负责把时间步文本转换为 `float | None`。
- 若时间步为空或格式异常，则回退为默认读取。

本阶段数据流设计：
- 用户选择时间步 `0.5` -> `_selected_visualization_time()` -> `0.5`。
- `_load_selected_field_surface()` -> `OpenFoamVtkService.build_geometry_filter(project, time_value=0.5)`。
- `OpenFoamVtkService` -> `vtkOpenFOAMReader.SetTimeValue(0.5)`。
- `vtkCompositeDataGeometryFilter.UpdateTimeStep(0.5)` -> 输出该时间步数据。
- UI 根据字段 `p` 或 `U` 绘制表面图。

本阶段核心业务场景完成进度：
- 可视化业务场景：时间步选择器已从 UI 展示升级为真实数据读取参数。
- 字段选择业务场景：继续支持 `p` 和 `U`。
- Case 隔离业务场景：仍读取当前 case。
- 未完成内容：单元字段显示、时间步动画播放、速度箭头和流线。
