# Sprint 5：求解运行页增强

当前阶段目标：
- 把“求解运行”页从占位文本改成真实任务面板。
- 让用户在运行前能看到当前项目、Case 路径、命令和参数摘要。
- 本阶段不做残差曲线、不做结果可视化、不做复杂错误智能诊断。

本阶段完成功能：
- 求解运行页新增真实运行摘要。
- 展示当前项目名称。
- 展示当前 Case 路径。
- 展示执行命令：`blockMesh && icoFoam`。
- 展示参数摘要：`endTime`、`deltaT`、`writeInterval`、`nu`。
- 新增页面按钮：
  - `运行最小仿真`
  - `停止当前任务`
  - `刷新摘要`
- 项目切换、参数加载、参数保存、运行开始、运行完成、运行失败、停止任务时会刷新运行页状态。

本阶段页面与 UI 完成情况：

求解运行页 `MainWindow._build_solver_run_tab()`：
- 页面标题：`求解运行`。
- UI 组件：`QLabel`、`QPushButton`、`QTextEdit`。
- 按钮或菜单项：
  - `运行最小仿真`：调用 `_run_minimal_simulation()`。
  - `停止当前任务`：调用 `_stop_current_process()`。
  - `刷新摘要`：调用 `_refresh_solver_run_panel()`。
- 键盘操作：本阶段未新增快捷键。
- 鼠标操作：点击按钮即可运行、停止或刷新摘要。
- 状态提示：显示空闲、运行中、完成、失败、停止等状态。

本阶段完成设计：
- UI 层新增求解运行页构建函数 `_build_solver_run_tab()`。
- 新增 `_refresh_solver_run_panel()`，统一刷新运行页摘要。
- 运行页摘要通过 `case_parameter_service.load()` 读取真实 OpenFOAM 参数。
- 运行状态仍由 `QProcess` 生命周期驱动，不引入新的任务队列。

本阶段数据流设计：
- 用户选择项目 -> 加载参数 -> 刷新求解运行页摘要。
- 用户保存参数 -> 写入 OpenFOAM 字典 -> 刷新求解运行页摘要。
- 用户点击运行 -> 保存参数 -> 切换到求解运行页 -> 状态改为运行中 -> 执行 `blockMesh && icoFoam`。
- `QProcess.finished` -> 根据退出码更新求解运行页状态。
