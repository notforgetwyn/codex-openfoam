## 阶段 35：失败步骤识别 + 一键仿真流水线

当前阶段目标：
- 同时完成两个能力：前处理失败步骤识别，以及“前处理完成后运行最小求解”的一键仿真流水线。
- 本阶段只做步骤标记、失败定位和串联执行，不做自动修复、不做复杂任务队列、不做求解器选择器。

本阶段完成功能：
- 几何/CAD 页新增 `一键仿真流水线` 按钮。
- 顶部 `几何/CAD` 菜单新增 `一键仿真流水线` 菜单项。
- `一键前处理` 命令链增加 `FOAMDESK_STEP` 步骤标记。
- `一键仿真流水线` 会自动执行：生成 snappy 配置 -> `blockMesh` -> `snappyHexMesh -overwrite` -> `checkMesh` -> `icoFoam`。
- 一键仿真会先保存当前参数配置，再运行 OpenFOAM 命令。
- 一键仿真成功后会导出求解指标、绘制残差曲线、刷新结果索引。
- 一键前处理或一键仿真失败时，会根据最后一个 `FOAMDESK_STEP` 标记提示失败步骤。
- 当前可识别失败步骤：`blockMesh 背景网格生成`、`snappyHexMesh 贴体网格生成`、`checkMesh 网格质量检查`、`icoFoam 最小求解`。

本阶段页面与 UI 完成情况：

几何/CAD 页 `GeometryTab`：
- 页面标题：`几何/CAD`，继续使用 `QLabel` 渲染。

  - `一键前处理`：执行 `blockMesh -> snappyHexMesh -> checkMesh`。
  - `一键仿真流水线`：执行 `blockMesh -> snappyHexMesh -> checkMesh -> icoFoam`。
  - 其他已有按钮保持不变。

- 鼠标操作：点击 `一键仿真流水线` 后，如果有多个 STL，会弹出 STL 选择框；如果只有一个 STL，会直接使用该 STL。

  - 运行中：`一键仿真流水线运行中。`

  - 失败诊断会额外显示 `失败步骤：xxx`。

本阶段完成设计：
- `MainWindow._run_simulation_pipeline()` 负责编排完整仿真流水线。
- `MainWindow._detect_failed_pipeline_step()` 根据 `FOAMDESK_STEP` 日志标记识别失败步骤。
- `_active_process_kind` 新增 `simulationPipeline`。
- `_process_label()` 新增 `simulationPipeline` 显示映射。
- 一键前处理命令链也加入步骤标记，失败定位能力同时覆盖前处理和一键仿真。
- 本阶段继续复用 `QProcess`，避免过早引入任务队列。

本阶段数据流设计：
- 用户点击 `一键仿真流水线` -> 检查项目/Case/OpenFOAM 环境 -> 保存参数。
- 选择 STL -> 读取 snappy 参数 UI -> 写入 `snappyHexMeshDict` 和 `snappy_config.json`。
- 启动 `QProcess` -> 执行 `blockMesh && snappyHexMesh -overwrite && checkMesh && icoFoam`。
- 每一步运行前输出 `FOAMDESK_STEP:步骤名`。
- stdout/stderr -> 底部日志面板。
- 成功结束 -> 导出求解指标 -> 绘制残差曲线 -> 刷新结果。
- 失败结束 -> 读取最后一个步骤标记 -> 写入问题/诊断面板。
