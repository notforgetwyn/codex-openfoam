## 阶段 17：速度矢量箭头 v1

当前阶段目标：
- 在结果页增加速度矢量箭头可视化能力，让用户能看到流体速度 `U` 的方向和大小。
- 本阶段只做稳定的采样箭头显示，不做复杂流线、切面矢量场、交互式探针和动画流线。
- 继续坚持 Python 内置可视化路线：VTK 负责读取 OpenFOAM 结果，Matplotlib 负责在 FoamDesk 内显示，不依赖 ParaView。

本阶段完成功能：
- 结果页新增 `加载速度箭头` 按钮。
- 点击 `加载速度箭头` 后，读取当前 Case 的 OpenFOAM 结果字段 `U`。
- 在独立 `FoamDesk 3D 视图` 窗口中绘制三维速度箭头。
- 箭头方向表示流体速度方向。
- 箭头颜色表示速度大小 `|U|`，旁边色条显示速度大小范围。
- 支持读取当前时间步选择器中的时间步，例如 `0.5`。
- 为避免界面卡顿，速度箭头会做采样显示，当前最多显示约 700 个箭头。
- 如果当前 Case 没有 `U` 字段，会给出错误提示并列出可用字段。

本阶段页面与 UI 完成情况：

结果页 `ResultsTab`：
- 页面标题：`结果`，继续使用 `QLabel` 渲染。
- UI 组件：`QPushButton`、`QComboBox`、`QSpinBox`、`QTextEdit`、`FigureCanvas`。
- 按钮或菜单项：
- `加载速度箭头`：可点击，触发 `_load_velocity_vectors()`，读取当前 Case 的 `U` 字段并打开 3D 视图。
- `字段` 下拉框：仍用于字段表面图和动画，本阶段不强制依赖它，速度箭头固定读取 `U`。
- `时间步` 下拉框：速度箭头会读取当前选中的时间步。
- 键盘操作：本阶段未新增键盘快捷键。
- 鼠标操作：点击 `加载速度箭头` 后弹出或刷新独立 3D 视图窗口。
- 状态提示：结果文本区会显示字段、时间步、箭头数量、速度范围和采样说明；状态栏显示 `速度箭头已加载。`

3D 视图 `VtkViewerDialog`：
- 页面标题：`Velocity Vectors |U|`。
- UI 组件：Matplotlib 3D 坐标轴、三维箭头、颜色条。
- 按钮或菜单项：无。
- 鼠标操作：可用 Matplotlib 默认交互查看图像窗口。
- 状态提示：无数据时显示 `No velocity vector data to display.`；速度全为 0 时显示 `Velocity field is zero everywhere.`。

本阶段完成设计：
- `VtkViewerDialog` 新增 `plot_velocity_vectors()`，专门负责速度矢量箭头绘制。
- `MainWindow` 新增 `_load_velocity_vectors()`，专门负责 UI 按钮、读取 OpenFOAM 数据、错误处理和结果反馈。
- 继续保持职责分离：
- UI 层负责按钮和反馈。
- OpenFOAM/VTK 服务负责读取真实 case。
- 3D 视图负责图像绘制。
- 本阶段没有把 VTK 直接嵌入 Qt 原生控件，避免 WSL 下 VTK Qt 控件透明、覆盖、卡顿的问题。
- 后续可以在同一设计上扩展速度切面、流线、粒子轨迹和矢量动画。

本阶段数据流设计：
- 用户点击 `加载速度箭头` -> `MainWindow._load_velocity_vectors()`。
- `_load_velocity_vectors()` -> 读取当前 Case 和当前时间步。
- `_load_velocity_vectors()` -> `OpenFoamVtkService.inspect()` 获取可用字段和时间步。
- `_load_velocity_vectors()` -> `OpenFoamVtkService.build_geometry_filter(project, time_value)` 读取真实 OpenFOAM 数据。
- VTK 输出 `poly_data` -> 读取点字段 `U`。
- `U` 字段 -> `VtkViewerDialog.plot_velocity_vectors()`。
- `plot_velocity_vectors()` -> `vtk_to_numpy()` 转为 NumPy 数组 -> 计算速度模长 `|U|` -> 采样 -> Matplotlib `quiver()` 绘制箭头。
- 绘图完成 -> 结果页文本区更新箭头数量和速度范围 -> 状态栏更新。
