## 阶段 13 修复：用 matplotlib 3D 替换 VTK 原生窗口，解决 3D 窗口空白

当前阶段目标：
- 修复独立 `FoamDesk 3D 视图` 窗口打开后只有白色/透明边框、没有三维内容的问题。
- 根因是 WSL GUI 环境下 VTK 原生 OpenGL 控件仍然无法稳定绘制。
- 本阶段不新增切面、流线和字段选择器，只保证当前 3D 验证场景、真实 case、压力云图能被用户看见。

本阶段完成功能：
- `FoamDesk 3D 视图` 不再使用 `QVTKRenderWindowInteractor` 作为界面显示控件。
- 改用 matplotlib 的 3D 画布 `FigureCanvasQTAgg` 承载三维图。
- `加载 3D 技术验证场景` 现在用 matplotlib 3D 绘制蓝色线框立方体。
- `加载真实 OpenFOAM 3D Case` 现在用 VTK 读取 OpenFOAM case，再把点数据转成 numpy，用 matplotlib 3D 绘制 case 点云。
- `加载压力云图` 现在用 VTK 读取压力字段 `p`，再用 matplotlib 3D 点云着色显示压力分布，并显示颜色条。
- 保留 VTK OpenFOAM Reader 作为数据读取层，但不再用 VTK 原生控件做 GUI 渲染层。

本阶段页面与 UI 完成情况：

页面名 `ResultsTab`：
- 页面标题：`结果`。
- UI 组件：`QPushButton`、`QTextEdit`、`FigureCanvas`、提示 `QLabel`。
- 按钮或菜单项：
  - `加载 3D 技术验证场景`：打开独立窗口，显示线框立方体。
  - `加载真实 OpenFOAM 3D Case`：打开独立窗口，显示真实 case 采样点云。
  - `加载压力云图`：打开独立窗口，显示压力 `p` 点云云图。
- 键盘操作：本阶段没有新增快捷键。
- 鼠标操作：点击三个可视化按钮都会打开或激活 `FoamDesk 3D 视图`。
- 状态提示：结果页文本区、状态栏、日志面板都会显示当前加载状态。

页面名 `VtkViewerDialog`：
- 页面标题：`FoamDesk 3D 视图`。
- UI 组件：`QDialog`、`FigureCanvasQTAgg`、matplotlib 3D axes。
- 按钮或菜单项：本窗口暂不提供工具栏按钮。
- 键盘操作：本阶段没有新增快捷键。
- 鼠标操作：matplotlib 3D 画布支持基础旋转、缩放和平移交互。
- 状态提示：主窗口负责显示加载状态。

本阶段完成设计：
- OpenFOAM 数据读取仍由 VTK 完成，保持对 OpenFOAM case 格式的兼容。
- GUI 显示改为 matplotlib 3D，避开 WSL 下 VTK 原生 OpenGL 控件透明、空白、覆盖 Qt 页面的问题。
- `VtkViewerDialog` 虽然名字暂时保留，但职责已经变为“Python 3D 可视化窗口”。
- 当前真实 case 和压力云图采用点云方式显示，后续再逐步增强为切面、表面、流线和矢量图。

本阶段数据流设计：
- 用户点击 `加载 3D 技术验证场景` -> `VtkViewerDialog.plot_cube()` -> matplotlib 3D 绘制线框立方体。
- 用户点击 `加载真实 OpenFOAM 3D Case` -> `OpenFoamVtkService.build_geometry_filter()` -> VTK 输出 `poly_data` -> `vtk_to_numpy` 转换点坐标 -> matplotlib 3D 绘制点云。
- 用户点击 `加载压力云图` -> VTK 读取 `p` 字段 -> `vtk_to_numpy` 转换点坐标和压力数组 -> matplotlib 3D scatter 按压力着色 -> 显示颜色条。
- 如果当前压力范围为常量 -> 仍会扩展显示范围，避免颜色条退化。
