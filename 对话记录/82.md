## 阶段 48：非盒状计算域模板

当前阶段目标：
- 增加不只是正方体/长方体的计算域模板。
- 让计算域预览图按真实模板顶点显示，而不是所有模板都画成长方体。
- 本阶段仍使用 OpenFOAM `blockMesh` 可直接生成的六面体块，不引入 CAD 曲面建模。

本阶段完成功能：
- 新增 `中等：渐扩风洞 6 x 2.5 x 2.5`。
- 新增 `高等：斜坡通道 8 x 3 x 2.5`。
- `ComputationDomainTemplate` 新增 `shape` 字段。
- `ProjectService.domain_vertices()` 根据模板形状生成 8 个 blockMesh 顶点。
- `blockMeshDict` 生成逻辑改为使用模板顶点。
- 几何/CAD 页计算域预览改为显示模板真实线框。
- 一键导入模板 STL 支持新模板：
  - 渐扩风洞默认导入 `medium_cylinder_obstacle.stl`
  - 斜坡通道默认导入 `medium_ramp_wedge.stl`

本阶段页面与 UI 完成情况：

几何/CAD 页：
- 页面标题：`几何/CAD`。
- UI 组件：`QComboBox`、`FigureCanvas`、`QPushButton`。
- 按钮或菜单项：
  - `计算域模板`：新增渐扩风洞和斜坡通道选项。
  - `一键导入模板 STL`：可按新模板导入匹配 STL。
- 键盘操作：本阶段未新增快捷键。
- 鼠标操作：用户选择新模板后，计算域预览线框会按非盒状形态显示。
- 状态提示：沿用已有模板说明和日志。

本阶段完成设计：
- 当前非盒状计算域仍保持单个 `hex` block，保证 `blockMesh` 可直接生成。
- 渐扩风洞通过入口截面小、出口截面大的 8 点六面体实现。
- 斜坡通道通过底面倾斜的 8 点六面体实现。
- 预览图和 `blockMeshDict` 共用同一套顶点生成逻辑，避免“显示的”和“实际生成的”不一致。

本阶段数据流设计：
- 用户选择模板 -> `ProjectService.domain_vertices()` -> UI 预览画线框。
- 用户应用模板 -> `ProjectService._block_mesh_dict_for_template()` -> 写入 `system/blockMeshDict`。
- 用户一键导入模板 STL -> 按模板 key 选择示例 STL -> 导入当前 Case。

本阶段核心业务场景完成进度：
- 计算域业务场景：从盒状模板扩展到非盒状模板。
- 前处理业务场景：新增渐扩风洞和斜坡通道入口。
- STL 测试业务场景：新模板可复用已有圆柱和楔形 STL。
- 求解业务场景：未变更。
- 未完成内容：圆管计算域、弯管计算域、O-grid 翼型外流域、真实曲面 CAD 计算域留到后续阶段。
