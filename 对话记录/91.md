## 阶段 73：删除旧求解器准备页并合并边界表页面

当前阶段目标：
- 删除旧的 `求解器准备` 页面。
- 将原来的 `仿真准备` 页面改名为 `求解器准备` 页面。
- 保留 patch 边界条件表，让用户继续在求解器准备页配置 `0/U`、`0/p` 和 `physicalProperties`。
- 修复一键启动时 `_snappy_min_refinement_input` 不存在导致的崩溃。

本阶段完成功能：
- 删除旧 `求解器准备` tab 入口。
- 工作区页面顺序调整为：`项目主页 -> 绘制几何 -> 求解器准备 -> 求解器选择 -> 仿真参数 -> 求解运行 -> 环境检查 -> 设置 -> 结果`。
- 原 `仿真准备` 页面改名为 `求解器准备`。
- 求解器准备页保留 `Patch 边界条件表`。
- 求解器准备页继续支持 `刷新求解器准备状态`、`补齐边界和物性`、`一键启动仿真`。
- `_read_snappy_settings()` 增加默认回退：如果 snappy 控件不存在，会读取已有配置；没有配置则使用 `SnappyHexMeshSettings()` 默认值。
- 一键启动不再因为缺少 `_snappy_min_refinement_input` 崩溃。

本阶段页面与 UI 完成情况：

求解器准备页 `SolverPrepareTab`：
- 页面标题：`求解器准备`。
- UI 组件：`QPushButton`、`QTextEdit`、`QTableWidget`、`QVBoxLayout`、`QHBoxLayout`。
- 按钮或菜单项：
  - `刷新求解器准备状态`：刷新 `0/U`、`0/p`、`physicalProperties` 状态和 patch 表格。
  - `补齐边界和物性`：按表格写入 OpenFOAM 字段文件。
  - `进入求解器选择`、`进入仿真参数`、`进入求解运行`：继续作为流程跳转。
  - `一键启动仿真`：继续调用统一仿真流水线。
- 键盘操作：本阶段未新增快捷键。
- 鼠标操作：用户可编辑 patch 表格并点击按钮生成文件。
- 状态提示：页面文案统一为 `求解器准备`，不再显示旧 `仿真准备` 页面名。

本阶段完成设计：

本阶段数据流设计：
- 用户绘制几何 -> 生成 `blockMeshDict + STL`。
- 用户进入 `求解器准备` -> 表格读取 patch -> 用户编辑 `U/p` 边界条件。
- 用户点击 `补齐边界和物性` -> 写入 `0/U`、`0/p`、`physicalProperties` 和 `foamdesk_boundary_table.json`。
- 用户点击 `一键启动仿真` -> 自动读取/生成 snappy 设置 -> 应用边界表 -> 执行 OpenFOAM 流水线。
