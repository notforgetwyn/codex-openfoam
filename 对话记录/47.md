## 阶段 15 修复：切换 Case 时清空旧结果缓存

当前阶段目标：
- 修复切换到未运行过的 case 后，界面仍显示上一个 case 残差图和指标的问题。
- 明确区分“磁盘结果隔离”和“界面内存缓存隔离”。
- 本阶段不改变 OpenFOAM 求解流程，只修复 UI 结果缓存。

本阶段完成功能：
- 切换 case 时清空当前进程输出 `_current_process_output`。
- 切换 case 时清空关键指标摘要。
- 切换 case 时清空失败诊断摘要。
- 切换 case 时清空残差曲线图。
- 如果当前 case 没有 `foamdesk_results/residuals.csv`，残差图显示 `Current case has no residual data.`。
- 当前 case 没有结果时，任务面板显示 `当前项目暂无求解结果`。
- 继续只读取当前 case 下的 `foamdesk_results`，不读取旧的项目级 `results`。

本阶段页面与 UI 完成情况：

页面名 `MainWindow`：
- 页面标题：`FoamDesk`。
- UI 组件：`Case 树`、结果页、求解运行页、底部任务面板。
- 按钮或菜单项：
  - 点击 `Case 树` 中不同 case，会先清理旧 case 的运行缓存。
  - 点击 `结果` 页，只看到当前 case 的结果索引。
  - 点击 `绘制残差曲线`，只读取当前 case 的 `foamdesk_results/residuals.csv`。
- 鼠标操作：
  - 从 `case` 切换到 `132` 后，如果 `132` 没运行过，不再显示 `case` 的残差曲线。
- 状态提示：
  - 无结果 case 显示 `暂无求解结果`。

本阶段完成设计：
- 新增 `_clear_case_runtime_state()`，统一清理 case 相关运行时 UI 缓存。
- `_activate_project()` 在切换当前 project/case 后立即调用 `_clear_case_runtime_state()`。
- `_restore_project_result_state()` 在当前 case 无结果时再次清理残差图，防止旧图残留。
- 结果文件仍按上一阶段设计保存在 `case/foamdesk_results`。

本阶段数据流设计：
- 用户点击 `132` case -> `_activate_project(132)`。
- `_activate_project()` -> `_clear_case_runtime_state()` -> 清空旧运行输出、指标、诊断、残差图。
- `_refresh_results_panel()` -> 读取 `132` case 的 OpenFOAM 时间步和字段。
- `_restore_project_result_state()` -> 检查 `132/foamdesk_results`。
- 如果 `132/foamdesk_results/residuals.csv` 不存在 -> 显示无残差数据，不再使用 `case` 的旧曲线。

本阶段核心业务场景完成进度：
- Case 管理业务场景：切换 case 后 UI 状态隔离完成。
- 结果管理业务场景：当前 case 结果读取和旧结果清理完成。
- 求解运行业务场景：未变更。
- 可视化业务场景：未变更。
- 未完成内容：旧项目级 `results` 目录迁移或清理、每个 case 的运行历史列表。
