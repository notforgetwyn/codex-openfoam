## 阶段 40：计算域模板与内置 STL 示例

当前阶段目标：
- 给用户提供简单、中等、高等三个计算域模板，让用户不用手写 `blockMeshDict`。
- 给每个计算域配套可测试的 STL 示例文件。
- 本阶段只做固定模板选择，不做任意尺寸自定义计算域、不做复杂边界条件编辑。

本阶段完成功能：
- 新增 3 个计算域模板：
  - `简单：单位立方体 1 x 1 x 1`，网格 `10 x 10 x 10`。
  - `中等：小型风洞 4 x 2 x 2`，网格 `40 x 20 x 20`。
  - `高等：长风洞 10 x 4 x 4`，网格 `80 x 32 x 32`。
- 几何/CAD 页新增 `计算域模板` 下拉框。
- 几何/CAD 页新增 `应用计算域模板` 按钮。
- 应用模板后会重写当前 Case 的 `system/blockMeshDict`。
- 应用模板后会同步 `0/U` 和 `0/p` 的边界条件，避免边界名不匹配。
- 应用模板后会把建议的 `locationInMesh` 填入 snappyHexMesh 参数区。
- 新增 4 个内置 ASCII STL 示例：
  - `simple_center_cube.stl`
  - `medium_cylinder_obstacle.stl`
  - `medium_ramp_wedge.stl`
  - `advanced_simplified_vehicle.stl`
- 更新 `assets/test_geometries/README.md`，说明每个 STL 适合哪个计算域。

本阶段页面与 UI 完成情况：

几何/CAD 页：
- 页面标题：`几何/CAD`。
- UI 组件：`QComboBox`、`QPushButton`、`QLabel`、`QFormLayout`。
- 按钮或菜单项：
  - `计算域模板`：选择简单、中等、高等计算域。
  - `应用计算域模板`：把所选模板写入当前 Case。
  - `导入 STL`：继续用于导入 STL 几何。
  - `生成 snappyHexMeshDict`：继续使用当前 STL 和 snappy 参数生成配置。
- 键盘操作：本阶段未新增快捷键。
- 鼠标操作：用户选择模板后点击应用，页面会刷新模板说明和建议参数。
- 状态提示：底部状态栏显示“计算域模板已应用”，日志面板记录模板尺寸、网格和建议 `locationInMesh`。

本阶段完成设计：
- `ProjectService` 新增 `ComputationDomainTemplate` 数据结构。
- `ProjectService.domain_templates()` 集中维护模板定义。
- `ProjectService.apply_domain_template()` 负责写入 `blockMeshDict`、同步边界字段、保存 `domain_config.json`。
- 计算域模板统一采用风洞式边界：
  - 左侧 `inlet`
  - 右侧 `outlet`
  - 其余面 `fixedWalls`
- 新建 Case 的默认 blockMeshDict 也切换为模板生成方式，避免旧 `movingWall/fixedWalls` 和新流程不一致。

本阶段数据流设计：
- 用户在几何/CAD 页选择计算域模板 -> 点击应用 -> `ProjectService.apply_domain_template()` -> 写入 `system/blockMeshDict` -> 同步 `0/U`、`0/p` -> 写入 `system/domain_config.json` -> UI 填入建议 `locationInMesh`。
- 用户选择对应 STL -> 导入 STL -> 生成 snappyHexMeshDict -> blockMesh 使用当前计算域生成背景网格 -> snappyHexMesh 在该计算域内对 STL 贴体。
- 内置 STL 文件不自动导入，用户仍通过“导入 STL”选择文件，避免误改当前 Case。

本阶段核心业务场景完成进度：

- STL 测试业务场景：完成简单、中等、高等配套测试 STL。
- OpenFOAM 集成业务场景：完成模板到 `blockMeshDict` 的生成链路。
- snappyHexMesh 业务场景：完成模板建议 `locationInMesh` 自动填充。
- 主菜单业务场景：未变更。
- 结果可视化业务场景：未变更。
