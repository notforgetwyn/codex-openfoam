# Sprint 3 小修复：旧项目最小仿真模板兼容

当前阶段目标：
- 修复旧项目运行最小仿真时报 `system/controlDict` 缺失的问题。
- 修复旧项目 `blockMeshDict` 边界名与 `0/U`、`0/p` 边界字段不一致的问题。
- 本阶段只做旧项目兼容，不进入参数配置页开发。

本阶段完成功能：
- 运行最小仿真前自动补齐旧项目缺失的最小 OpenFOAM 文件。
- 自动补齐 `0/U`、`0/p`、`constant/transportProperties`、`constant/physicalProperties`、`system/controlDict`、`system/fvSchemes`、`system/fvSolution`。
- 自动解析 `system/blockMeshDict` 的 `boundary` 区域。
- 如果 `0/U`、`0/p` 的边界字段和 `blockMeshDict` 不一致，自动按实际边界名重写字段文件。
- 支持旧模板边界名 `inlet/outlet/walls`。

本阶段页面与 UI 完成情况：

主窗口 `MainWindow`：
- 页面标题：保持 `FoamDesk`。
- UI 组件：运行按钮、求解运行页、底部日志/任务面板保持不变。
- 按钮或菜单项：点击“运行”前会自动执行旧项目 case 兼容修复。
- 键盘操作：本阶段未新增键盘操作。
- 鼠标操作：点击“运行”后仍进入最小仿真流程。
- 状态提示：如果补齐了旧项目文件，日志面板会输出“已补齐旧项目缺失的最小仿真文件”。

本阶段完成设计：
- `ProjectService.ensure_minimal_case_template()` 不再只是补缺文件，还会同步 `U/p` 的边界字段。
- `ProjectService` 增加 `blockMeshDict` 边界解析逻辑，使用括号深度扫描，避免被 `faces ((...));` 误截断。
- UI 层不直接处理 OpenFOAM 字典细节，只调用项目服务完成 case 准备。

本阶段数据流设计：
- 用户点击“运行” -> `MainWindow._run_minimal_simulation()` -> `ProjectService.ensure_minimal_case_template()`。
- `ensure_minimal_case_template()` -> 补齐缺失文件 -> 解析 `blockMeshDict` 边界名 -> 检查 `0/U` 和 `0/p` -> 必要时重写字段文件。
- case 准备完成 -> 执行 `blockMesh && icoFoam` -> 日志面板显示 OpenFOAM 输出。
