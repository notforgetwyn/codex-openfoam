## 阶段 14 修复：Case 树只显示当前项目 Case，并关闭旧主窗口

当前阶段目标：
- 修复 `Case 树` 只是改名但仍然列出所有项目的问题。
- 让 `Case 树` 只显示当前项目下的 case。
- 修复返回项目选择时旧主工作台仍留在后面的问题。

本阶段完成功能：
- `Case 树` 不再列出所有项目。
- `Case 树` 现在只显示当前项目和当前项目下的 `case`。
- 点击 `case` 只选中当前项目的 case，不再触发项目切换。
- 返回项目选择时，旧 `MainWindow` 会先关闭。
- 选择新项目后创建新的 `MainWindow`。
- 取消项目选择后，也会按原项目创建新的 `MainWindow`，恢复用户原来的工作台。
- 去掉对 `setWindowOpacity()` 的依赖，避免 WSL Qt 后端不支持透明度导致警告。

本阶段页面与 UI 完成情况：

页面名 `MainWindow`：
- 页面标题：`FoamDesk`。
- UI 组件：顶部菜单栏、左侧 `Case 树`、右侧工作区、底部日志面板。
- `Case 树` 显示结构：

```text
Case 树
└─ 123
   └─ case
```

- 按钮或菜单项：
  - `文件 -> 返回项目选择`：关闭旧主窗口，打开项目选择页。
  - `项目 -> 返回项目选择`：关闭旧主窗口，打开项目选择页。
  - `项目 -> 刷新 Case 树`：刷新当前项目下的 case 显示。
- 鼠标操作：
  - 点击 `case` 会在日志中输出当前 Case 路径。
  - 不再把所有项目都显示在 Case 树里。
- 状态提示：
  - 点击 case 后状态栏显示 `Case 已选中。`

页面名 `StartupWindow`：
- 页面标题：`FoamDesk - 选择项目`。
- UI 组件：项目列表、新建项目、打开项目、刷新。
- 鼠标操作：
  - 选择项目后创建新主窗口。
  - 取消选择后恢复原项目主窗口。

本阶段完成设计：
- `Case 树` 的职责重新限定为“当前项目内部 case 导航”。
- 项目选择职责只放在 `StartupWindow`。
- `_refresh_project_tree()` 改为读取 `self._current_project`，而不是 `ProjectService.list_projects()`。
- `_on_project_tree_item_clicked()` 改为只选中当前 case，不再调用 `open_project()`。
- `_return_to_project_selection()` 改为先关闭旧窗口，再打开项目选择页，避免旧主窗口留在后面。

本阶段数据流设计：
- 当前项目激活 -> `_refresh_project_tree()` -> 左侧只渲染当前项目和 `case`。
- 点击 `case` -> 输出当前 case 路径 -> 状态栏显示已选中。
- 点击返回项目选择 -> 关闭旧 `MainWindow` -> 打开 `StartupWindow`。
- 选择项目 -> 创建新 `MainWindow(context, initial_project=selected_project)`。
- 取消选择 -> 创建新 `MainWindow(context, initial_project=current_project)`。
