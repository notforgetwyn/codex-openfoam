# Sprint 4：参数配置页接入真实 OpenFOAM 字典文件

当前阶段目标：
- 把“参数配置”页从占位文本改成真实可用表单。
- 先接入最小仿真最核心的 `controlDict` 与 `physicalProperties` 参数。
- 本阶段不做复杂求解器选择、边界条件编辑、网格参数编辑和结果可视化。

本阶段完成功能：
- 参数配置页新增 4 个可编辑参数：
  - `endTime`：仿真结束时间。
  - `deltaT`：时间步长。
  - `writeInterval`：结果写出间隔。
  - `nu`：运动粘度。
- 支持“加载当前项目参数”。
- 支持“保存参数到 Case”。
- 支持“恢复默认参数”。
- 新建、打开、点击项目树项目后，会自动加载当前项目参数。
- 点击“运行”前会先保存参数，参数非法时不会启动 OpenFOAM。
- 保存参数时同时更新：
  - `system/controlDict`
  - `constant/physicalProperties`
  - `constant/transportProperties`

本阶段页面与 UI 完成情况：

参数配置页 `MainWindow._build_parameter_tab()`：
- 页面标题：`参数配置`。
- UI 组件：`QLineEdit`、`QSpinBox`、`QPushButton`、`QLabel`、`QTextEdit`。
- 按钮或菜单项：
  - `加载当前项目参数`：读取当前 case 文件并填充表单。
  - `保存参数到 Case`：校验参数并写入 OpenFOAM 字典文件。
  - `恢复默认参数`：恢复默认值到表单，但不立即写入文件。
- 键盘操作：本阶段未新增快捷键。
- 鼠标操作：用户可点击输入框编辑参数，点击按钮加载/保存/恢复默认。
- 状态提示：参数加载、保存、恢复默认后显示当前状态；错误会进入底部“问题”面板。

本阶段完成设计：
- 新增 `SimulationParameters` 领域模型，用于承载可编辑仿真参数。
- 新增 `OpenFoamCaseParameterService`，负责读写 OpenFOAM 字典文件。
- `ApplicationContext` 注入参数服务，UI 通过服务读写，不直接散落解析逻辑。
- UI 层只负责表单展示、输入收集、状态提示和错误提示。
- 参数服务负责文件解析、参数替换、默认值和合法性校验。

本阶段数据流设计：
- 用户选择项目 -> `ProjectService.open_project()` -> `OpenFoamCaseParameterService.load()` -> 参数表单显示。
- 用户点击保存 -> UI 读取表单 -> 组装 `SimulationParameters` -> `OpenFoamCaseParameterService.save()` -> 写入 OpenFOAM 字典。
- 用户点击运行 -> 先保存表单参数 -> 保存成功 -> 执行 `blockMesh && icoFoam`。
- 参数文件读写：
  - `system/controlDict`：读写 `endTime/deltaT/writeInterval`。
  - `constant/physicalProperties`：读写 `nu`。
  - `constant/transportProperties`：同步写入 `nu`，兼容旧版本。
