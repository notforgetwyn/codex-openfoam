## 阶段 29：STL 生成 snappyHexMeshDict

当前阶段目标：
- 把 Sprint 28 的 STL 导入能力继续向 OpenFOAM 前处理推进，让导入的 STL 能生成 `system/snappyHexMeshDict`。
- 本阶段只做配置文件生成，不直接运行 `snappyHexMesh`，避免还没有边界配置、网格质量检查和失败诊断时把流程做复杂。

本阶段完成功能：
- 几何/CAD 页新增 `生成 snappyHexMeshDict` 按钮。
- 顶部 `几何/CAD` 菜单新增 `生成 snappyHexMeshDict` 菜单项。
- 当前 Case 有 1 个 STL 时，点击后直接生成 `system/snappyHexMeshDict`。
- 当前 Case 有多个 STL 时，弹出选择框，让用户选择用于 `snappyHexMesh` 的 STL。
- 生成完成后，几何清单会显示 `snappyHexMeshDict` 路径。
- 同时写入 `constant/triSurface/snappy_config.json`，记录使用的 STL、字典路径和生成时间。

本阶段页面与 UI 完成情况：

几何/CAD 页 `GeometryTab`：
- 页面标题：`几何/CAD`，继续使用 `QLabel` 渲染。
- UI 组件：继续使用 `QPushButton`、`QTextEdit` 和顶部 `QMenu`。
- 按钮或菜单项：
  - `导入 STL`：选择本地 STL，并复制到当前 Case 的 `constant/triSurface`。
  - `刷新几何清单`：重新读取当前 Case 的几何清单和 snappy 字典状态。
  - `生成 snappyHexMeshDict`：根据已导入 STL 生成 OpenFOAM 网格配置字典。
  - `预览已导入 STL`：在独立 3D 视图窗口预览 STL。
  - `STEP/IGES 支持说明`：说明复杂 CAD 格式为什么需要后续接入 OCCT/CAD 内核。
- 键盘操作：本阶段未新增快捷键。
- 鼠标操作：点击 `生成 snappyHexMeshDict` 后，如果有多个 STL，会先选择几何文件；如果只有一个 STL，会直接生成配置文件。
- 状态提示：成功后状态栏显示 `snappyHexMeshDict 生成完成。`，日志区记录生成路径。

本阶段完成设计：
- `GeometryImportService` 新增 `generate_snappy_hex_mesh_dict()`，负责从已导入 STL 生成 OpenFOAM 字典。
- `GeometryImportService` 新增 `_snappy_hex_mesh_dict()`，集中维护 MVP 版 snappyHexMesh 配置模板。
- `MainWindow` 只负责 UI 交互和用户选择，不直接拼 OpenFOAM 字典内容。
- 本阶段保持“服务层生成文件、UI 层触发操作”的边界，避免 OpenFOAM 字典逻辑散落在界面事件里。
- 生成的字典采用保守默认参数：开启 `castellatedMesh` 和 `snap`，暂不开启边界层 `addLayers`。

本阶段数据流设计：
- 用户导入 STL -> `GeometryImportService.import_stl()` -> 复制到 `constant/triSurface` -> 更新 `geometry_manifest.json`。
- 用户点击生成 -> `MainWindow._generate_snappy_hex_mesh_dict()` -> 读取当前 Case 的 STL 清单 -> 调用 `GeometryImportService.generate_snappy_hex_mesh_dict()`。
- 服务生成文件 -> 写入 `system/snappyHexMeshDict` -> 写入 `constant/triSurface/snappy_config.json` -> UI 刷新几何清单。
- 本阶段不运行 OpenFOAM 命令，不修改 `0/U`、`0/p`、`controlDict`、`fvSchemes`、`fvSolution`。

- Case 管理业务场景：继续支持当前 Case 隔离，生成的字典只写入当前 Case。
- 求解业务场景：未接入 `snappyHexMesh` 执行，本阶段只完成运行前配置文件生成。
- 结果业务场景：未变更。
- 几何/CAD 业务场景：从“STL 导入和预览”推进到“STL 驱动 snappyHexMesh 配置生成”。
- 可视化业务场景：未变更。
