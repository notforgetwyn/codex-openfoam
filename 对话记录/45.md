## 阶段 15 修复：新增 Case 功能与多 Case 树

当前阶段目标：
- 在当前项目内支持新增 Case。
- 让左侧 `Case 树` 真正显示当前项目下的多个 case。
- 选择不同 case 后，主工作台切换到该 case 的参数、求解和结果上下文。

本阶段完成功能：
- `Case` 菜单新增 `新增 Case`。
- 点击后弹出 Case 名称输入框。
- 创建新的 case 目录，例如 `case_2`。
- 新 case 自动生成最小 OpenFOAM 模板文件。
- 创建完成后自动切换到新 case。
- `Case 树` 会列出当前项目下所有有效 case。
- 当前选中的 case 后面显示 `✓`。
- 点击 `Case 树` 中的 case 可以切换当前 case。

本阶段页面与 UI 完成情况：

页面名 `MainWindow`：
- 页面标题：`FoamDesk`。
- UI 组件：顶部菜单栏、左侧 `Case 树`、工作区、底部日志面板。
- 按钮或菜单项：
  - `Case -> 新增 Case`：创建当前项目下的新 case。
  - `Case -> 打开当前 Case 目录`：保留。
  - `项目 -> 刷新 Case 树`：刷新当前项目下 case。
- 鼠标操作：
  - 点击 `Case 树` 中的 case，会切换当前 case。
  - 当前 case 显示为 `case_name ✓`。
- 状态提示：
  - 创建成功后显示 `Case 创建完成。`
  - 切换成功后显示 `Case 已切换。`

本阶段完成设计：
- `SimulationProject` 新增 `case_name` 字段。
- `ProjectService` 新增 `list_cases(project)`。
- `ProjectService` 新增 `create_case(project, name)`。
- `ProjectService` 新增 `switch_case(project, case_name)`。
- `Case 树` 不再只固定显示 `case`，而是扫描当前项目目录下所有有效 case。
- 当前 `SimulationProject.case_dir` 始终指向当前选中的 case。

本阶段数据流设计：
- 用户点击 `Case -> 新增 Case` -> 输入名称 -> `ProjectService.create_case()` -> 创建目录和模板文件 -> `_activate_project()` 切换到新 case。
- 用户点击 `Case 树` 中的某个 case -> `ProjectService.switch_case()` -> 返回新的 `SimulationProject` -> `_activate_project()` -> 加载参数、刷新求解页、刷新结果页。
- 当前 case 的求解和结果继续使用 `SimulationProject.case_dir`。

本阶段核心业务场景完成进度：
- Case 管理业务场景：完成新增 case 和切换 case。
- 项目管理业务场景：未变更。
- 参数配置业务场景：切换 case 后自动加载该 case 参数。
- 求解运行业务场景：运行时针对当前 case 执行。
- 结果管理业务场景：切换 case 后刷新当前 case 结果索引。
- 未完成内容：Case 删除、复制、重命名、设置默认 case、每个 case 独立运行历史。
