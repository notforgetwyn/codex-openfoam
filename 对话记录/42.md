## 阶段 14 修复：顶部主题菜单与项目选择隐藏修正

当前阶段目标：
- 把主题切换入口从工具栏/设置页补充到顶部菜单栏。
- 修复点击返回项目选择时主工作台没有立即隐藏的问题。
- 本阶段不改变求解、项目数据和可视化逻辑。

本阶段完成功能：
- 顶部菜单栏新增 `主题` 菜单。
- `主题` 菜单中列出所有当前主题名称。
- 用户可以直接点击具体主题进行切换。
- `主题` 菜单保留 `循环切换主题` 入口。
- 点击 `项目选择` 或 `返回项目选择` 时，主窗口先隐藏并强制处理 Qt 事件，再打开项目选择页。
- 取消项目选择后主窗口恢复显示。
- 选择项目后切换项目并恢复主窗口。

本阶段页面与 UI 完成情况：

页面名 `MainWindow`：
- 页面标题：`FoamDesk`。
- UI 组件：`QMenuBar`、项目树、工作区、底部日志面板。
- 按钮或菜单项：
  - 新增顶部 `主题` 菜单。
  - `主题 -> vscode-dark` 等具体主题项可直接切换主题。
  - `主题 -> 循环切换主题` 保留。
  - `文件 -> 返回项目选择` 继续可用。
  - `项目 -> 返回项目选择` 继续可用。
- 键盘操作：本阶段没有新增快捷键。
- 鼠标操作：点击主题菜单项立即应用主题并保存设置。
- 状态提示：主题切换后状态栏显示 `主题已切换为 ...`。

页面名 `StartupWindow`：
- 页面标题：`FoamDesk - 选择项目`。
- UI 组件：项目列表、新建项目、打开项目、刷新。
- 鼠标操作：主窗口隐藏后显示项目选择页。
- 状态提示：未变更。

本阶段完成设计：
- 新增 `_set_theme(theme_name)`，用于按指定主题名切换主题。
- `_cycle_theme()` 改为复用 `_set_theme()`，避免主题切换逻辑重复。
- `_return_to_project_selection()` 在 `self.hide()` 后调用 `QApplication.processEvents()`，确保主窗口隐藏事件立即生效。

本阶段数据流设计：
- 用户点击顶部 `主题` 菜单项 -> `_set_theme(theme_name)` -> 更新 `settings.yaml` -> `_apply_settings_theme()` -> 刷新环境面板 -> 状态栏提示。
- 用户点击返回项目选择 -> `MainWindow.hide()` -> `QApplication.processEvents()` -> `StartupWindow.exec()`。
- 用户取消选择 -> `MainWindow.show()`。
- 用户选择项目 -> `_activate_project()` -> `MainWindow.show()`。
