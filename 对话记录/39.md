## 阶段 14 修复：主工作台返回项目选择与项目结果记忆

当前阶段目标：
- 让用户在主工作台界面可以回到项目选择阶段。
- 让每个项目保留自己的已有结果状态，避免用户误以为每次打开项目都要重新求解。
- 明确规则：打开项目只读取已有文件，不自动重新运行 OpenFOAM；只有点击 `运行` 才会重新求解。

本阶段完成功能：
- 顶部 `文件` 菜单新增 `返回项目选择`。
- 顶部 `项目` 菜单新增 `返回项目选择`。
- 工具栏新增 `项目选择`。
- 点击后打开 `FoamDesk - 选择项目` 启动页。
- 在项目选择页选择项目后，主工作台切换到该项目。
- 切换项目后自动保存为上次项目。
- 激活项目时自动读取该项目已有结果索引。
- 如果项目已有时间步、网格、残差 CSV 或指标 JSON，会在任务面板显示“已加载项目已有结果”。
- 如果项目已有 `results/residuals.csv`，会自动绘制残差曲线。
- 如果项目没有求解结果，会显示“当前项目暂无求解结果”。

本阶段完成设计：
- `MainWindow` 新增 `_return_to_project_selection()`，用于从主工作台返回项目选择页。
- `MainWindow` 新增 `_restore_project_result_state()`，用于激活项目后恢复已有结果状态。
- `_activate_project()` 中统一调用 `_restore_project_result_state()`。
- 项目记忆仍由 `ProjectService.remember_project()` 统一处理。
- 求解仍然只由 `_run_minimal_simulation()` 触发，打开项目和切换项目不会自动运行 OpenFOAM。

本阶段数据流设计：
- 用户点击 `项目选择` -> `StartupWindow` -> 用户选择项目 -> `_activate_project()`。
- `_activate_project()` -> `ProjectService.remember_project()` -> 写入 `last_project_path`。
- `_activate_project()` -> `ResultIndexService.index()` -> 读取已有时间步、网格和字段文件。
- `_restore_project_result_state()` -> 检查 `results/residuals.csv` 和 `results/metrics.json`。
- 如果 `residuals.csv` 存在 -> `_plot_residual_curve()` -> 绘制已有残差曲线。
- 没有任何结果 -> 任务面板提示当前项目暂无求解结果。
