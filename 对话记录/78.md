## 阶段 41：计算域预览与一键导入模板 STL

当前阶段目标：
- 在几何/CAD 页显示计算域和 STL 的相对位置，让用户直观看到 STL 是否放在计算域内。
- 增加“一键导入模板 STL”，让用户选择计算域后能快速导入匹配示例几何。
- 本阶段不做任意尺寸计算域编辑、不做拖拽移动 STL、不做多模板批量导入。

本阶段完成功能：
- 几何/CAD 页新增计算域预览图。
- 预览图中灰色线框表示当前计算域。
- 预览图中蓝色半透明几何表示当前 Case 已导入的 STL。
- 切换计算域模板时，预览图会刷新计算域尺寸。
- 当前 Case 导入 STL 后，刷新几何清单会同步刷新预览图。
- 新增 `一键导入模板 STL` 按钮。
- 一键导入会先应用当前计算域模板，再导入匹配的内置 STL。
- 简单模板默认导入 `simple_center_cube.stl`。
- 中等模板默认导入 `medium_cylinder_obstacle.stl`。
- 高等模板默认导入 `advanced_simplified_vehicle.stl`。

本阶段页面与 UI 完成情况：

几何/CAD 页：
- 页面标题：`几何/CAD`。
- UI 组件：`QComboBox`、`QPushButton`、`QLabel`、`FigureCanvas`。
- 按钮或菜单项：
  - `计算域模板`：选择简单、中等、高等计算域。
  - `应用计算域模板`：只应用计算域，不导入 STL。
  - `一键导入模板 STL`：应用当前计算域，并导入对应示例 STL。
  - `刷新几何清单`：刷新文本清单和计算域预览。
- 键盘操作：本阶段未新增快捷键。
- 鼠标操作：用户选择模板后，预览图显示对应计算域；点击一键导入后，预览图显示导入后的 STL。
- 状态提示：日志会记录模板 STL 名称和同步的计算域模板。

本阶段完成设计：
- 计算域预览复用 Matplotlib 3D 画布，不引入新的前端依赖。
- STL 预览复用已有 VTK STL 读取逻辑，保证和导入预览使用同一套数据读取方式。
- 一键导入模板 STL 不直接运行 OpenFOAM，只准备 Case 几何和计算域，避免用户误触发长时间计算。
- 模板 STL 文件仍保存在 `assets/test_geometries`，导入时复制到当前 Case 的 `constant/triSurface`。

本阶段数据流设计：
- 用户选择计算域模板 -> `_refresh_domain_template_hint()` -> `_refresh_domain_preview()` -> 绘制计算域线框。
- 用户点击一键导入模板 STL -> `ProjectService.apply_domain_template()` -> `GeometryImportService.import_stl()` -> `_refresh_geometry_panel()` -> `_refresh_domain_preview()`。
- 预览数据流：当前 Case -> `GeometryImportService.list_assets()` -> VTK 读取 STL 顶点和三角面 -> Matplotlib 绘制 STL 与计算域线框。

本阶段核心业务场景完成进度：
- 前处理业务场景：完成计算域与 STL 相对位置预览。
- 模板测试业务场景：完成当前计算域的一键 STL 导入。
- OpenFOAM Case 准备业务场景：完成“计算域 + STL”快速准备入口。
- 求解业务场景：未变更。
- 结果可视化业务场景：未变更。
- 未完成内容：计算域尺寸自定义、STL 导入后位置编辑、预览中拖拽移动 STL、多 STL 勾选显示留到后续阶段。
