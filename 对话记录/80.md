## 阶段 44/45/46：几何页自适应、STL 旋转与边界条件配置

当前阶段目标：
- 修复几何/CAD 页在窗口缩放后组件互相挤压的问题。
- 增加 STL 旋转功能，让用户可以在导入或编辑 STL 时设置绕 X/Y/Z 轴旋转角度。
- 增加边界条件配置 UI，让用户能设置入口速度、出口压力和壁面类型。
- 本阶段不做鼠标拖拽、不做 STL 批量编辑、不做复杂多物理边界条件。

本阶段完成功能：
- 几何/CAD 页外层改为可滚动页面。
- 顶部操作按钮拆成两行，避免按钮挤压。
- 自定义计算域输入拆成尺寸行和网格行。
- 计算域预览图和几何清单文本区设置最小高度，避免互相覆盖。
- `StlTransform` 新增 `rotate_degrees`。
- STL 导入/编辑弹窗新增 `旋转 X° / 旋转 Y° / 旋转 Z°`。
- STL 变换顺序明确为：缩放 -> 旋转 -> 平移。
- STL manifest 新增旋转角度记录。
- 几何/CAD 页新增边界条件配置：
  - 入口速度 `Ux/Uy/Uz`
  - 出口压力 `p`
  - 壁面类型 `noSlip/slip`
- 新增 `应用边界条件` 按钮。
- 应用边界条件后会写入当前 Case 的 `0/U`、`0/p` 和 `system/boundary_config.json`。

本阶段页面与 UI 完成情况：

几何/CAD 页：
- 页面标题：`几何/CAD`。
- UI 组件：`QScrollArea`、`QDoubleSpinBox`、`QSpinBox`、`QComboBox`、`QPushButton`、`FigureCanvas`、`QTextEdit`。
- 按钮或菜单项：
  - `导入 STL`：导入前可设置缩放、旋转、平移。
  - `编辑 STL 位置`：导入后可重新设置缩放、旋转、平移。
  - `应用边界条件`：写入入口速度、出口压力和壁面类型。
  - `应用自定义计算域`：保持可用。
- 键盘操作：本阶段未新增快捷键。
- 鼠标操作：用户通过滚动查看全部几何设置；通过数值输入控制 STL 旋转和边界条件。
- 状态提示：日志记录 STL transform、边界条件写入文件和状态栏结果。

STL 变换弹窗：
- 页面标题：`导入 STL：位置和缩放`。
- UI 组件：scale、平移 X/Y/Z、旋转 X/Y/Z、实时 3D 预览。
- 按钮或菜单项：
  - `OK`：确认当前变换并写入 STL。
  - `Cancel`：取消，不改 Case。
- 状态提示：预览区显示旋转后的 STL 位置。

本阶段完成设计：
- STL 旋转在 `GeometryImportService` 中实现，避免 UI 直接处理几何算法。
- 坐标变换统一封装为 `_transform_vertex()`，导入、编辑和预览使用一致逻辑。
- 边界条件配置封装到 `ProjectService.apply_boundary_conditions()`，UI 不直接拼 OpenFOAM 字典。
- 边界条件配置持久化到 `system/boundary_config.json`，再次打开 Case 时能加载回 UI。
- 几何页通过 `QScrollArea` 保证小窗口下组件可访问。

本阶段数据流设计：
- 用户导入/编辑 STL -> 输入 scale/rotate/translate -> 预览刷新 -> 确认 -> `GeometryImportService` 重写 STL -> 更新 `geometry_manifest.json`。
- 用户设置边界条件 -> 点击应用 -> `ProjectService.apply_boundary_conditions()` -> 写入 `0/U`、`0/p`、`system/boundary_config.json`。
- 用户缩放窗口 -> `QScrollArea` 保证页面内容可滚动访问，不改变业务数据。

本阶段核心业务场景完成进度：
- STL 管理业务场景：完成导入和导入后旋转编辑。
- 前处理业务场景：完成基础边界条件配置入口。
- UI 可用性场景：完成几何/CAD 页缩放自适应修复。
- OpenFOAM Case 准备业务场景：完成入口速度、出口压力、壁面类型写入。
- 求解业务场景：未变更。
- 未完成内容：鼠标拖拽 STL、旋转中心选择、多 STL 批量编辑、更多边界类型、湍流/温度边界配置。
