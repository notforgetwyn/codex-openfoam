## 阶段 62：求解器选择页与仿真参数页

当前阶段目标：
- 新增独立的 `求解器选择` 页面，让用户能选择 `icoFoam`、`simpleFoam`、`pisoFoam`。
- 将原来的 `参数配置` 页面升级为 `仿真参数` 页面，覆盖时间、写出、收敛、流体属性和基础数值设置。
- 本阶段先完成页面、配置保存和核心 OpenFOAM 字典写入，不做复杂多物理场和完整不同求解器运行流水线。

本阶段完成功能：
- 工作区新增 `求解器选择` 标签页。
- `求解器选择` 页面支持选择：
  - `icoFoam`：入门不可压瞬态流。
  - `simpleFoam`：稳态不可压流，适合风洞/绕流。
  - `pisoFoam`：瞬态不可压流。
- `求解器选择` 页面支持选择湍流模型、数值格式、fvSolution 预设。
- `仿真参数` 页面新增最大迭代次数、收敛残差、流体密度 rho。
- 保存参数时会写入：
  - `system/controlDict`
  - `system/fvSchemes`
  - `system/fvSolution`
  - `constant/physicalProperties`
  - `constant/transportProperties`
  - `system/foamdesk_simulation_config.json`
- 求解运行页参数摘要新增 solver、rho、残差、湍流模型、fvSchemes、fvSolution 信息。

本阶段页面与 UI 完成情况：

求解器选择页 `SolverSelectTab`：
- 页面标题：`求解器选择`。
- UI 组件：`QComboBox`、`QPushButton`、`QLabel`、`QTextEdit`。
- 按钮或菜单项：
  - `加载当前 Case 求解器`：读取当前 Case 配置并回填页面。
  - `保存求解器配置`：保存 solver、湍流模型、数值格式和求解设置。
- 键盘操作：本阶段未新增快捷键。
- 鼠标操作：选择下拉框后点击保存即可写入当前 Case。
- 状态提示：页面底部显示当前加载/保存的求解器配置。

仿真参数页 `SimulationParameterTab`：
- 页面标题：`仿真参数`。
- UI 组件：`QLineEdit`、`QSpinBox`、`QPushButton`、`QLabel`、`QTextEdit`。
- 按钮或菜单项：
  - `加载当前项目参数`：读取 `controlDict`、物性文件和 FoamDesk 配置 JSON。
  - `保存参数到 Case`：写入 OpenFOAM 相关字典。
  - `恢复默认参数`：恢复默认值，保存后才生效。
- 键盘操作：本阶段未新增快捷键。
- 鼠标操作：用户可编辑表单后点击保存。
- 状态提示：保存后显示 solver、endTime、deltaT、writeInterval、rho、nu。

本阶段完成设计：
- 扩展 `SimulationParameters`，从只包含基础时间参数扩展为完整 MVP 仿真配置对象。
- 扩展 `OpenFoamCaseParameterService`，集中负责读取、校验、保存仿真参数。
- 用 `system/foamdesk_simulation_config.json` 保存 OpenFOAM 字典无法稳定表达的工程化配置。
- `controlDict` 负责 solver、endTime、deltaT、writeInterval。
- `fvSchemes` 和 `fvSolution` 根据用户选择的预设生成。
- 先保持页面和配置解耦于具体求解执行，后续再让一键仿真流水线读取 solver 配置。

本阶段数据流设计：
- 用户进入 `求解器选择` -> 选择 solver/湍流/数值格式/求解预设 -> 点击保存 -> `_save_case_parameters()` -> `OpenFoamCaseParameterService.save()` -> 写入 OpenFOAM 字典和 FoamDesk JSON。
- 用户进入 `仿真参数` -> 编辑 endTime/deltaT/writeInterval/maxIterations/residual/rho/nu -> 点击保存 -> 写入 `controlDict`、物性文件、`fvSchemes`、`fvSolution`。
- 打开项目或切换 Case -> `_load_case_parameters()` -> 从当前 Case 读取配置 -> 回填两个页面。
- 求解运行页刷新 -> `case_parameter_service.load()` -> 展示完整参数摘要。
