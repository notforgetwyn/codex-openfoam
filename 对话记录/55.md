## 阶段 20.1：结果页操作区下拉菜单化

当前阶段目标：
- 解决结果页顶部按钮过多、横向占用空间太大、界面拥挤的问题。
- 本阶段只调整按钮组织方式，不删除功能、不改变后端计算逻辑。
- 保留字段选择、时间步选择、播放控制等连续操作控件。

本阶段完成功能：
- 将结果页第一行大量按钮合并为 4 个下拉菜单。
- 新增 `结果数据` 下拉菜单，包含：`刷新结果索引`、`导出求解指标`、`绘制残差曲线`。
- 新增 `3D 视图` 下拉菜单，包含：`加载 3D 技术验证场景`、`加载真实 OpenFOAM 3D Case`。
- 新增 `云图` 下拉菜单，包含：`加载压力云图`、`加载压力表面云图`、`加载所选字段表面图`。
- 新增 `速度场` 下拉菜单，包含：`加载速度箭头`、`加载速度切面`、`加载速度流线`。
- 第二行仍保留 `字段`、`时间步`、`刷新可视化字段`、`上一帧`、`下一帧`、`播放动画`、`暂停动画`。

本阶段页面与 UI 完成情况：

结果页 `ResultsTab`：
- 页面标题：`结果`，未变化。
- UI 组件：`QPushButton`、`QMenu`、`QComboBox`、`QSpinBox`、`QTextEdit`、`FigureCanvas`。
- 按钮或菜单项：
- `结果数据`：点击后展开结果数据类操作。
- `3D 视图`：点击后展开基础 3D 加载操作。
- `云图`：点击后展开压力云图和字段表面图操作。
- `速度场`：点击后展开速度箭头、速度切面、速度流线操作。
- 键盘操作：本阶段未新增键盘快捷键。
- 鼠标操作：点击菜单按钮会下拉显示具体功能项；点击菜单项触发原有业务逻辑。
- 状态提示：原有各功能的结果区和状态栏提示保持不变。

本阶段完成设计：
- 在 `_build_results_tab()` 内新增局部辅助函数 `make_menu_button()`，用于创建带下拉菜单的按钮。
- 原先直接铺开的 10 个按钮改为按业务类型分组。
- 业务函数未改动，菜单项仍调用原来的 `_refresh_results_panel()`、`_load_velocity_streamlines()` 等方法。
- 这种改法风险较低，因为只是 UI 入口重组，不影响数据读取、求解、可视化算法。

本阶段数据流设计：
- 用户点击 `速度场` -> 选择 `加载速度流线` -> 调用原有 `_load_velocity_streamlines()`。
- 用户点击 `云图` -> 选择 `加载压力表面云图` -> 调用原有 `_load_pressure_surface_cloud()`。
- 用户点击 `结果数据` -> 选择 `绘制残差曲线` -> 调用原有 `_plot_residual_curve()`。
- 所有菜单项只是入口变化，后续数据流与之前一致。

本阶段核心业务场景完成进度：
- 项目选择业务场景：未变更。
- Case 管理业务场景：未变更。
- 求解业务场景：未变更。
- 结果业务场景：结果页操作入口更清晰、更省空间。
- 可视化业务场景：云图、箭头、切面、流线功能全部保留，只是入口移动到菜单内。
- 未完成内容：字段选择和动画控制区仍然偏宽，后续可以继续做响应式布局或折叠面板。
