## 阶段 13 修复：3D 视图窗口化，解决结果页卡住和页面透穿

当前阶段目标：
- 修复结果页点击 `加载 3D 技术验证场景`、`加载真实 OpenFOAM 3D Case`、`加载压力云图` 后页面看起来卡住、结果页和设置页内容重叠的问题。
- 本阶段不新增新的 CFD 后处理能力，只修复可视化交互和窗口承载方式。
- 保持压力云图、真实 case 读取、3D 技术验证场景的原有数据逻辑不变。

本阶段完成功能：
- 移除结果页内部嵌入的 `QVTKRenderWindowInteractor`。
- 新增独立的 `FoamDesk 3D 视图` 窗口承载 VTK 三维渲染。
- 点击 `加载 3D 技术验证场景` 后，会打开或复用独立 3D 窗口显示蓝色线框立方体和坐标轴。
- 点击 `加载真实 OpenFOAM 3D Case` 后，会打开或复用独立 3D 窗口显示真实 case 几何。
- 点击 `加载压力云图` 后，会打开或复用独立 3D 窗口显示压力 `p` 标量云图和颜色标尺。
- 结果页继续显示文字反馈，说明当前加载内容、字段、时间步和压力范围。
- 规避 WSL GUI 环境下 VTK 原生控件嵌入 Qt Tab 时出现的层级覆盖、重绘错乱和页面透穿问题。

本阶段页面与 UI 完成情况：

页面名 `ResultsTab`：
- 页面标题：`结果`，使用 `QLabel` 渲染。
- UI 组件：`QPushButton`、`QTextEdit`、`FigureCanvas`、提示 `QLabel`。
- 按钮或菜单项：
  - `加载 3D 技术验证场景`：点击后打开独立 `FoamDesk 3D 视图`。
  - `加载真实 OpenFOAM 3D Case`：点击后读取当前项目 case，并在独立窗口中显示。
  - `加载压力云图`：点击后读取字段 `p`，并在独立窗口中显示云图。
- 键盘操作：本阶段没有新增快捷键。
- 鼠标操作：点击三个可视化按钮都会打开或激活独立 3D 窗口。
- 状态提示：
  - 结果页文本区显示当前正在加载或已加载的可视化内容。
  - 状态栏显示对应加载完成状态。
  - 日志面板继续记录加载动作。

页面名 `VtkViewerDialog`：
- 页面标题：`FoamDesk 3D 视图`。
- UI 组件：`QDialog`、`QVTKRenderWindowInteractor`、`vtkRenderer`。
- 按钮或菜单项：本窗口暂不提供工具栏按钮，只作为 3D 渲染承载窗口。
- 键盘操作：保留 VTK 默认交互。
- 鼠标操作：支持 VTK 默认旋转、缩放、平移交互。
- 状态提示：状态仍回写到主窗口结果页、状态栏和日志面板。

本阶段完成设计：
- 将 VTK 原生三维控件从 Qt Tab 页面中解耦，改为独立 `QDialog`。
- `MainWindow` 通过 `_ensure_vtk_viewer()` 创建或复用三维视图窗口。
- `MainWindow` 通过 `_finalize_vtk_render()` 统一完成渲染、显示、置顶。
- 结果页只负责操作入口和文字反馈，不再承载 VTK 原生窗口。
- 这个设计更适合 WSL GUI，因为 VTK 的 OpenGL/native widget 在复杂 Tab 布局里容易出现层级问题。

本阶段数据流设计：
- 用户点击 `加载 3D 技术验证场景` -> `MainWindow._ensure_vtk_viewer()` -> 创建 `VtkViewerDialog` -> VTK 生成测试立方体 -> 独立窗口渲染。
- 用户点击 `加载真实 OpenFOAM 3D Case` -> 当前项目 -> `OpenFoamVtkService.inspect()` -> `OpenFoamVtkService.build_geometry_filter()` -> 独立窗口渲染真实 case 几何。
- 用户点击 `加载压力云图` -> 当前项目 -> VTK reader 读取字段数组 -> 选择字段 `p` -> `vtkPolyDataMapper` 做标量着色 -> 独立窗口渲染云图。
- 主窗口结果页只显示文字反馈，不再参与 OpenGL 绘制。
