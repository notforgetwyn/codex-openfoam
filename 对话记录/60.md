## 阶段 24.1：结果页字段/切面/动画控制区菜单化

当前阶段目标：
- 解决结果页第二行控件横向铺得太长、视觉拥挤的问题。
- 保留所有原功能，只调整 UI 组织方式。
- 让第二行控件风格与 `结果数据 / 3D 视图 / 云图 / 速度场` 一致。

本阶段完成功能：
- 将原来横向铺开的 `字段`、`时间步`、`刷新可视化字段`、`加载所选字段表面图` 收纳到 `字段时间` 下拉菜单。
- 将原来横向铺开的 `切面` 和 `位置` 收纳到 `切面设置` 下拉菜单。
- 将原来横向铺开的 `上一帧`、`下一帧`、`帧间隔`、`播放动画`、`暂停动画` 收纳到 `动画控制` 下拉菜单。
- 原有功能全部保留，入口只是从平铺控件改为菜单内控件。
- 结果页顶部现在更紧凑，第一行是业务动作菜单，第二行是控制类菜单。

本阶段页面与 UI 完成情况：

结果页 `ResultsTab`：
- 页面标题：`结果`，未变化。
- UI 组件：`QPushButton`、`QMenu`、`QWidgetAction`、`QComboBox`、`QDoubleSpinBox`、`QSpinBox`。
- 按钮或菜单项：
- `字段时间`：包含字段选择、时间步选择、刷新字段、加载所选字段表面图。
- `切面设置`：包含切面方向和切面位置。
- `动画控制`：包含上一帧、下一帧、帧间隔、播放动画、暂停动画。
- 键盘操作：菜单展开后仍可在输入框中输入数值。
- 鼠标操作：点击菜单按钮展开控件面板，点击其中按钮执行原功能。
- 状态提示：各功能原有状态栏和结果区提示保持不变。

本阶段完成设计：
- 新增局部辅助函数 `make_widget_menu_button()`。
- 使用 `QWidgetAction` 将复杂控件面板嵌入 `QMenu`。
- 保留原有控件实例变量，例如 `_visual_field_combo`、`_visual_time_combo`、`_slice_axis_combo`、`_slice_position_input`。
- 因此原业务逻辑不需要大改，仍然从这些控件读取当前值。

本阶段数据流设计：
- 用户点击 `字段时间` -> 修改字段/时间步 -> 点击 `加载所选字段表面图` -> 原 `_load_selected_field_surface()` 数据流不变。
- 用户点击 `切面设置` -> 修改切面方向和位置 -> 点击 `速度场 -> 加载连续速度切面` -> 原切面数据流不变。
- 用户点击 `动画控制` -> 设置帧间隔 -> 点击播放动画 -> 原 `_play_visualization_animation()` 数据流不变。
