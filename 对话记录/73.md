## 阶段 36：流水线失败步骤中文修复建议

当前阶段目标：
- 在 Sprint 35 已能识别失败步骤的基础上，为每类失败步骤补充用户能看懂的中文修复建议。
- 本阶段只做静态规则建议，不做自动修复、不做 AI 诊断、不做日志深度分类。

本阶段完成功能：
- 一键前处理和一键仿真失败时，问题/诊断面板会先显示 `失败步骤：xxx`。
- 失败步骤后新增 `修复建议`。
- 支持 `blockMesh` 失败建议。
- 支持 `snappyHexMesh` 失败建议。
- 支持 `checkMesh` 失败建议。
- 支持 `icoFoam` 失败建议。
- 支持未知步骤的通用建议。

本阶段页面与 UI 完成情况：

问题/诊断面板 `ProblemPanel`：
- 页面标题：沿用底部 `问题` 标签页。
- UI 组件：继续使用 `QTextEdit` 展示诊断文本。
- 按钮或菜单项：本阶段未新增按钮。
- 键盘操作：本阶段未新增快捷键。
- 鼠标操作：用户点击一键前处理或一键仿真失败后，可在底部 `问题` 标签查看建议。
- 状态提示：
  - 失败时状态栏仍显示 `一键前处理失败，退出码 X` 或 `一键仿真流水线失败，退出码 X`。
  - 问题面板显示失败步骤和对应修复建议。

本阶段完成设计：
- 新增 `MainWindow._pipeline_step_advice()`。
- `_detect_failed_pipeline_step()` 继续负责识别失败步骤。
- `_on_process_finished()` 在失败时把失败步骤、修复建议和原有日志诊断合并显示。
- 建议逻辑先采用静态规则，避免过早引入复杂日志分类系统。

本阶段数据流设计：
- OpenFOAM 命令失败 -> `_on_process_finished(exit_code != 0)`。
- `_detect_failed_pipeline_step()` 从日志中读取最后一个 `FOAMDESK_STEP`。
- `_pipeline_step_advice()` 根据失败步骤返回中文建议。
- 诊断文本合并为：失败步骤 -> 修复建议 -> 原有 OpenFOAM 日志诊断。
- 合并后的文本写入底部 `问题` 面板。
