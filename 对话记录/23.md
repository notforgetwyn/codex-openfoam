# Sprint 3 小修复：环境检查按钮启动异常修复

当前阶段目标：
- 修复启动后点击或刷新环境检查时出现的 Traceback。
- 本阶段只修复 UI 信号参数问题，不扩展新的 OpenFOAM 功能。

本阶段完成功能：
- 修复“环境检查”刷新按钮把 Qt 的 `checked: bool` 参数误传给环境状态刷新函数的问题。
- 环境刷新函数现在会忽略 `bool` 参数并重新调用环境检测器。
- 程序启动后不再因 `bool` 没有 `is_available` 字段而报错。

本阶段页面与 UI 完成情况：

设置页 / 环境检查区域 `MainWindow`：
- 页面标题：保持现有设置页和环境检查区域。
- UI 组件：保持刷新按钮、环境状态文本框。
- 按钮或菜单项：“刷新环境检查”可点击，触发真实环境检测。
- 键盘操作：本阶段未新增键盘操作。
- 鼠标操作：点击刷新按钮后更新环境检查结果。
- 状态提示：环境检查完成后继续在日志区显示 OpenFOAM 状态。

本阶段完成设计：
- 修正 Qt 信号与业务方法的边界：按钮点击信号通过 `lambda` 调用业务刷新方法，不直接把 Qt 内部参数传入业务参数。
- `_refresh_environment_panels()` 增加防御逻辑，如果收到 `bool`，自动重新检测环境。

本阶段数据流设计：
- 用户点击刷新按钮 -> Qt `clicked(bool)` -> `lambda` 丢弃 bool -> `_refresh_environment_panels()`。
- `_refresh_environment_panels()` -> `environment_detector.detect()` -> 返回 `OpenFOAMEnvironmentStatus` -> 更新环境文本和日志。

本阶段核心业务场景完成进度：

- OpenFOAM 执行业务场景：未变更。
- 项目管理业务场景：未变更。
- UI 组件化业务场景：增强按钮信号接入的稳定性。
