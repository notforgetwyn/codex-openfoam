## 阶段 38：STL 位置变换与 3D 视图范围过滤

当前阶段目标：
- 让用户导入 STL 时可以设置位置和大小，不再完全依赖 STL 文件原始坐标。
- 在独立 3D 视图窗口中增加显示范围设置，支持查看全部数据、所有 STL 附近数据、单个 STL 附近数据。
- 同时纳入前面已验证的 snappy 几何语法修复和 `importedGeometry` 边界同步修复。

本阶段完成功能：
- 导入 STL 时新增位置/缩放对话框。
- 支持设置 `scale` 缩放比例。
- 支持设置 STL 平移 `X/Y/Z`。
- 非默认变换会直接改写导入到 Case 的 STL 顶点坐标。
- STL 导入 manifest 中记录 `transform.translate` 和 `transform.scale`。
- 独立 3D 视图窗口新增显示范围控件：
  - `全部显示`
  - `所有 STL 附近`
  - `单个 STL 附近`
- 独立 3D 视图窗口新增 STL 下拉框和附近半径输入。
- 点云、压力点云、速度箭头、速度切面、连续速度切面、表面图会按该范围过滤显示。
- 修复 OpenFOAM-dev `snappyHexMeshDict` 几何块，改为 `file "xxx.stl";` 写法。
- 一键仿真前会同步 `importedGeometry` 到 `0/U` 和 `0/p`，避免 `icoFoam` 边界不匹配。

本阶段页面与 UI 完成情况：

STL 导入对话框 `ImportStlTransformDialog`：
- 页面标题：`导入 STL：位置和缩放`。
- UI 组件：`QDialog`、`QFormLayout`、`QDoubleSpinBox`、`QDialogButtonBox`。
- 按钮或菜单项：`OK` 确认导入，`Cancel` 取消导入。
- 键盘操作：支持在数值框中直接输入 scale 和 X/Y/Z。
- 鼠标操作：支持点击上下调节数值。
- 状态提示：对话框提示这些参数会直接修改导入后的 STL 顶点坐标。

独立 3D 视图窗口 `VtkViewerDialog`：
- 页面标题：`FoamDesk 3D 视图`。
- UI 组件：新增 `QComboBox` 和 `QDoubleSpinBox`。
- 按钮或菜单项：
  - `显示范围`：选择全部、所有 STL 附近、单个 STL 附近。
  - `STL`：选择某个已导入 STL。
  - `附近半径`：控制 STL 包围盒外扩范围。
- 键盘操作：未新增快捷键。
- 鼠标操作：用户在独立窗口中修改过滤范围后，再加载新的 3D 图即可按范围显示。
- 状态提示：如果当前范围没有数据，图中会显示“当前显示范围内没有数据”。

本阶段完成设计：
- 新增 `StlTransform` 数据类，表示 STL 导入时的平移和缩放。
- `GeometryImportService.import_stl()` 支持可选 `StlTransform`。
- `GeometryImportService._write_transformed_ascii_stl()` 负责 ASCII STL 顶点变换。
- `VtkViewerDialog` 维护当前 Case 的 STL bounds，用包围盒加半径做可视化过滤。
- 过滤设置放在独立 3D 窗口中，没有污染主菜单或结果页主控件。
- 过滤先做 MVP：基于 STL 包围盒附近区域，不做复杂几何距离计算。

本阶段数据流设计：
- 用户点击 `导入 STL` -> 选择文件 -> 输入 scale/X/Y/Z。
- `MainWindow._read_stl_transform_dialog()` -> 生成 `StlTransform`。
- `GeometryImportService.import_stl()` -> 复制或变换 ASCII STL -> 写入 `constant/triSurface`。
- `VtkViewerDialog.set_geometry_assets()` -> 读取当前 Case STL bounds。
- 用户在独立窗口选择显示范围 -> 加载压力/速度/表面/点云图 -> 绘图函数按 STL bounds 过滤点或面片。

本阶段核心业务场景完成进度：
- 几何/CAD 业务场景：STL 导入支持位置和缩放控制。
- 可视化业务场景：独立 3D 窗口支持基于 STL 附近区域过滤显示。
- 前处理业务场景：修复 OpenFOAM-dev snappy 几何语法。
- 求解业务场景：一键仿真前同步 snappy 后的 `importedGeometry` 边界，当前测试 Case 已验证 `icoFoam` 可运行到 `End`。
