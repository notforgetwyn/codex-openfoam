## 阶段 50：计算域预览状态与运行前几何诊断

当前阶段目标：
- 让用户明确知道几何/CAD 页当前计算域是“只在预览”还是“已经应用到当前 Case”。
- 增强运行前检查，提前发现 STL 与计算域边界距离过小、多个 STL 互相重叠、locationInMesh 放到 STL 内部等常见问题。
- 本阶段不改 OpenFOAM 求解器，不新增新的网格算法，只增强 UI 提示和诊断。

本阶段完成功能：
- 几何/CAD 页新增 `应用状态` 提示。
- 切换计算域下拉框后，3D 预览立即显示被选中的计算域模板。
- 如果只是切换下拉框但没有点击 `应用计算域模板`，界面会提示“预览中”，说明当前 Case 实际仍使用旧计算域。
- 点击 `应用计算域模板` 后，界面会提示“已应用”，说明该模板已经写入 OpenFOAM case。
- 运行前检查新增 STL 到计算域边界的最近距离诊断。
- 运行前检查新增 locationInMesh 是否可能落在 STL 固体内部的诊断。
- 运行前检查新增多个 STL 包围盒是否重叠，以及 STL 之间最近包围盒距离的诊断。

本阶段页面与 UI 完成情况：

几何/CAD 页 `GeometryTab`：
- 页面标题：`几何/CAD`，继续使用主工作区标签页展示。
- UI 组件：新增 `QLabel` 应用状态提示，继续使用 `QComboBox`、`QPushButton`、Matplotlib 3D Canvas。
- 按钮或菜单项：
  - `计算域模板` 下拉框：切换后只影响预览，不直接写入 case。
  - `应用计算域模板`：把当前下拉框模板写入 `system/blockMeshDict` 和 `system/domain_config.json`。
  - `运行前检查`：输出文件、边界、locationInMesh、STL 边界距离、STL 重叠等诊断。
- 键盘操作：本阶段未新增快捷键。
- 鼠标操作：鼠标选择不同计算域模板后，3D 预览区域立即刷新；点击应用后状态从预览变为已应用。
- 状态提示：绿色表示已应用，黄色表示仅预览。

本阶段完成设计：
- 增加 `_selected_domain_template()` 表示 UI 下拉框当前选择的模板。
- 保留 `_current_domain_template()` 表示当前 Case 已保存、已应用的模板。
- `_refresh_domain_preview()` 使用 `_selected_domain_template()`，让预览跟随用户选择。
- 运行前检查继续使用 `_current_domain_template()`，确保诊断基于实际 OpenFOAM case，而不是临时预览。
- 增加 `_update_domain_apply_state_label()`，集中维护“预览中/已应用”状态文案和颜色。

本阶段数据流设计：
- 用户切换计算域下拉框 -> `_refresh_domain_template_hint()` -> `_update_domain_apply_state_label()` -> `_refresh_domain_preview()` -> 3D 预览重绘。
- 用户点击 `应用计算域模板` -> `ProjectService.apply_domain_template()` -> 写入 `system/blockMeshDict` 和 `system/domain_config.json` -> `_refresh_geometry_panel()` -> 状态提示变为已应用。
- 用户点击 `运行前检查` -> `_current_domain_template()` 读取已应用模板 -> 读取 STL 顶点包围盒 -> 计算 STL 与计算域边界距离、locationInMesh 与 STL 包围盒关系、多个 STL 包围盒关系 -> 输出到问题面板和日志。
