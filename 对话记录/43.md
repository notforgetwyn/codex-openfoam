## 阶段 14 修复：Case 树命名与返回项目选择关闭主窗口

当前阶段目标：
- 将左侧树控件从“项目树”改名为更准确的“Case 树”。
- 修复返回项目选择时主工作台仍然显示在后面的问题。
- 本阶段不改变求解、结果、可视化和项目数据结构。

本阶段完成功能：
- 左侧树控件表头改为 `Case 树`。
- 菜单项 `刷新项目树` 改为 `刷新 Case 树`。
- 新手教程和阶段说明中的“项目树”文案改为 `Case 树`。
- 返回项目选择时，旧主窗口会先隐藏并变为透明。
- 用户选择新项目后，会创建新的 `MainWindow` 并关闭旧主窗口。
- 用户取消选择后，旧主窗口恢复显示。
- 避免旧主工作台继续留在项目选择窗口后面。

本阶段页面与 UI 完成情况：

页面名 `MainWindow`：
- 页面标题：`FoamDesk`。
- UI 组件：顶部菜单栏、左侧 `Case 树`、右侧工作区、底部日志面板。
- 按钮或菜单项：
  - `项目 -> 刷新 Case 树`。
  - `文件 -> 返回项目选择`。
  - `项目 -> 返回项目选择`。
- 键盘操作：本阶段没有新增快捷键。
- 鼠标操作：点击返回项目选择后进入项目选择阶段，旧主工作台不再留在后面。
- 状态提示：取消项目选择时显示 `已取消项目选择。`

页面名 `StartupWindow`：
- 页面标题：`FoamDesk - 选择项目`。
- UI 组件：项目列表、新建项目、打开项目、刷新。
- 鼠标操作：
  - 选择项目后关闭旧主窗口，打开新主窗口。
  - 取消选择后恢复旧主窗口。

本阶段完成设计：
- `_return_to_project_selection()` 中临时关闭 `quitOnLastWindowClosed`，避免关闭旧主窗口导致应用退出。
- 进入项目选择前对旧主窗口执行 `setWindowOpacity(0.0)` 和 `hide()`。
- 选择新项目后创建新的 `MainWindow(self._context, initial_project=...)`。
- 将新主窗口引用保存到 `QApplication` 对象上，避免被 Python 垃圾回收。
- 新窗口显示后关闭旧窗口。
- 取消选择时恢复旧窗口透明度和显示状态。

本阶段数据流设计：
- 点击返回项目选择 -> 旧 `MainWindow` 隐藏 -> `StartupWindow.exec()`。
- 取消选择 -> 旧 `MainWindow` 恢复。
- 选择项目 -> 创建新 `MainWindow` -> 传入 `initial_project` -> 新窗口激活项目并恢复已有结果 -> 关闭旧 `MainWindow`。
