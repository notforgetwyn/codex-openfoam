# Sprint 6：运行日志解析与失败诊断

当前阶段目标：
- 把 OpenFOAM 原始英文错误日志转换成新手能看懂的中文诊断。
- 失败时在底部“问题”面板和“求解运行”页展示诊断摘要。
- 本阶段只做常见错误识别，不做 AI 自动修复、不做复杂日志结构化数据库。

本阶段完成功能：
- 新增 OpenFOAM 日志诊断服务。
- 支持识别 `cannot find file` 缺文件错误。
- 支持识别 `Cannot find patchField entry` 边界条件名称不匹配。
- 支持识别 `Unable to set reference cell for field p` 压力参考值缺失。
- 支持识别 `FOAM FATAL ERROR` / `FOAM FATAL IO ERROR` 泛化致命错误。
- 运行失败后自动生成中文诊断。
- 诊断结果写入底部“问题”面板。
- 诊断结果同步显示在“求解运行”页的“最近诊断”区域。
- 正常完成时显示“本次任务正常完成，没有失败诊断”。

本阶段页面与 UI 完成情况：

求解运行页 `MainWindow._build_solver_run_tab()`：
- 页面标题：`求解运行`。
- UI 组件：新增 `最近诊断` 文本区域。
- 按钮或菜单项：运行、停止、刷新摘要保持原有逻辑。
- 键盘操作：本阶段未新增快捷键。
- 鼠标操作：点击运行后，如果失败，会在页面内看到中文诊断。
- 状态提示：运行中、完成、失败状态继续显示；失败时补充诊断文本。

底部问题面板：
- 失败时自动切换到“问题”面板。
- 显示退出码、诊断标题、原因和建议。

本阶段完成设计：
- 新增 `OpenFoamDiagnostic` 领域模型。
- 新增 `OpenFoamLogDiagnosticService` 服务，专门负责日志诊断。
- `ApplicationContext` 注入日志诊断服务。
- `MainWindow` 缓存本次 `QProcess` 的 stdout/stderr。
- `QProcess.finished` 失败时调用诊断服务生成摘要。
- UI 层不直接写具体 OpenFOAM 错误规则，规则集中在服务层，后续可扩展更多诊断。

本阶段数据流设计：
- OpenFOAM stdout/stderr -> `MainWindow._current_process_output`。
- `QProcess.finished(exit_code != 0)` -> `OpenFoamLogDiagnosticService.diagnose()`。
- 诊断列表 -> `format_diagnostics()` -> 中文摘要。
- 中文摘要 -> 底部“问题”面板。
- 中文摘要 -> “求解运行”页最近诊断区域。
