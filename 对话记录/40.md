## 阶段 14 修复：返回项目选择时隐藏主工作台

当前阶段目标：
- 修复用户从主工作台返回项目选择页时，主工作台仍然留在后面的问题。
- 让“返回项目选择”更像真正回到启动主菜单，而不是在主界面上叠一个选择窗口。
- 本阶段不改变项目记忆、求解和可视化逻辑。

本阶段完成功能：
- 点击 `项目选择`、`文件 -> 返回项目选择`、`项目 -> 返回项目选择` 后，主工作台会先隐藏。
- 用户在项目选择页选择项目后，主工作台重新显示并切换到所选项目。
- 用户取消项目选择后，主工作台重新显示，并保持原项目不变。
- 状态栏取消选择时仍显示 `已取消项目选择。`

本阶段页面与 UI 完成情况：

页面名 `MainWindow`：
- 页面标题：`FoamDesk`。
- UI 组件：菜单栏、工具栏、主工作台。
- 按钮或菜单项：
  - 工具栏 `项目选择`：隐藏主工作台，打开项目选择页。
  - `文件 -> 返回项目选择`：隐藏主工作台，打开项目选择页。
  - `项目 -> 返回项目选择`：隐藏主工作台，打开项目选择页。
- 键盘操作：本阶段没有新增快捷键。
- 鼠标操作：点击返回项目选择入口后，视觉上回到项目选择阶段。
- 状态提示：取消选择后提示 `已取消项目选择。`

页面名 `StartupWindow`：
- 页面标题：`FoamDesk - 选择项目`。
- UI 组件：项目列表、新建项目、打开项目、刷新。
- 按钮或菜单项：复用已有项目选择逻辑。
- 鼠标操作：选择项目后关闭启动页，主工作台重新显示。

本阶段完成设计：
- `_return_to_project_selection()` 进入项目选择前调用 `self.hide()`。
- 用户取消选择时调用 `self.show()`、`raise_()`、`activateWindow()` 恢复主工作台。
- 用户选择项目后先 `_activate_project()`，再恢复主工作台显示。
- 不关闭主窗口对象，只隐藏窗口，避免重建主窗口导致状态、服务和图形资源重复初始化。

本阶段数据流设计：
- 用户点击项目选择入口 -> `MainWindow.hide()` -> `StartupWindow.exec()`。
- 用户取消 -> `MainWindow.show()` -> 原项目保持不变。
- 用户选择项目 -> `_activate_project(selected_project)` -> 保存上次项目 -> 恢复已有结果状态 -> `MainWindow.show()`。

本阶段核心业务场景完成进度：
- 项目管理业务场景：返回项目选择体验修复完成。
- 项目记忆业务场景：未变更，仍保存最后选择项目。
- 求解运行业务场景：未变更，不会自动重算。
- 结果管理业务场景：未变更，切换项目后仍读取已有结果。
- 未完成内容：项目选择页搜索、最近项目排序、项目删除。
