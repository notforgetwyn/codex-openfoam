## 阶段 27：Markdown 报告 v2 自动嵌入图片

当前阶段目标：
- 将 Markdown 报告从纯文本汇总升级为带图片的报告。
- 导出报告时自动生成残差曲线图片，并自动导出当前 3D 视图窗口中已经打开的所有标签页图片。
- 本阶段只做 Markdown 报告增强，不做 PDF 导出、不做 Word 报告、不做自动工程结论生成。

本阶段完成功能：
- `导出 Markdown 报告` 现在会生成 `foamdesk_results/report_assets/` 目录。
- 如果当前 Case 有残差 CSV，导出报告时会自动生成 `residual_curve.png`。
- 如果 `FoamDesk 3D 视图` 窗口中已经打开了多个 3D 标签页，导出报告时会把这些标签页逐个保存为 PNG。
- `report.md` 从 `FoamDesk 仿真报告 v1` 升级为 `FoamDesk 仿真报告 v2`。
- `report.md` 新增 `报告图片` 章节，自动用 Markdown 图片语法引用生成的 PNG。
- `导出 PNG` 仍然保留手动导出当前 3D 标签页的能力。

本阶段页面与 UI 完成情况：

结果页 `ResultsTab`：
- 页面标题：`结果`，未变更。
- UI 组件：继续使用 `结果数据` 下拉菜单中的 `导出 Markdown 报告`。
- 按钮或菜单项：点击 `结果数据 -> 导出 Markdown 报告` 后，会自动生成报告图片和 `report.md`。
- 键盘操作：未新增快捷键。
- 鼠标操作：用户先打开压力云图、速度箭头、速度流线等 3D 标签页，再导出报告，报告会引用这些已打开视图。
- 状态提示：导出后结果文本提示报告已导出，并说明当前报告包含自动嵌入图片。

3D 视图窗口 `VtkViewerDialog`：
- 页面标题：`FoamDesk 3D 视图`，未变更。
- UI 组件：继续使用 `QTabWidget` 管理多个 3D 图。
- 新增内部能力：`export_all_pngs(output_dir)` 会导出当前所有 3D 标签页。
- 按钮或菜单项：用户界面上不新增按钮，报告导出流程内部调用该能力。

本阶段完成设计：
- `ReportExportService.export_markdown()` 增加 `asset_paths` 参数，用于接收报告图片路径。
- `ReportExportService` 负责把图片路径转换为相对 Markdown 链接，保证 `report.md` 在结果目录内可直接查看。
- `MainWindow._export_report_assets()` 负责生成报告素材，包括残差曲线和 3D 标签页图片。
- `MainWindow._export_residual_report_asset()` 使用独立 `Figure` 生成残差曲线 PNG，不依赖当前页面是否已经显示残差曲线。
- `VtkViewerDialog.export_all_pngs()` 遍历所有 3D 标签页，把每个 `FigureCanvas` 保存为 PNG。

本阶段数据流设计：
- 用户点击 `结果数据 -> 导出 Markdown 报告` -> `_export_markdown_report()`。
- `_export_markdown_report()` -> `_export_report_assets()` -> 创建 `foamdesk_results/report_assets/`。
- `_export_report_assets()` -> `_export_residual_report_asset()` -> 读取残差 CSV -> 保存 `residual_curve.png`。
- `_export_report_assets()` -> `VtkViewerDialog.export_all_pngs()` -> 导出当前所有 3D 标签页 PNG。
- `_export_markdown_report()` -> `ReportExportService.export_markdown(..., asset_paths)` -> 写入 `report.md`。
- `report.md` -> `报告图片` 章节 -> 引用 `report_assets/*.png`。
