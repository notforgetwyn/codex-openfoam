## 阶段 13：真实压力字段三维云图

当前阶段目标：
- 在结果页中基于 Python/VTK 读取 OpenFOAM 真实 case 字段，完成第一版压力 `p` 标量云图。
- 本阶段只做压力场三维表面着色，不做切面、流线、时间步切换动画和速度矢量箭头。
- 可视化必须嵌入 FoamDesk 客户端内部，不依赖 ParaView 等外部软件。

本阶段完成功能：
- 结果页新增 `加载压力云图` 按钮。
- 点击 `加载压力云图` 后，程序读取当前项目的 `case/foamdesk.foam` 标记文件。
- 通过 VTK 的 OpenFOAM Reader 加载真实 OpenFOAM case。
- 自动启用 OpenFOAM 输出中的点字段和单元字段数组。
- 自动识别压力字段 `p`，优先使用点字段 `p`，没有点字段时使用单元字段 `p`。
- 使用 `vtkPolyDataMapper` 对压力 `p` 做标量着色。
- 在三维视图中显示压力云图、坐标轴和颜色标尺。
- 如果当前 case 中没有压力字段 `p`，界面会弹出错误提示，并列出当前可用点字段和单元字段。
- 加载成功后，日志区会输出压力范围 `pRange` 和可用时间步。

  - `绘制残差曲线`：用 matplotlib 在客户端内部绘制残差曲线。
  - `加载 3D 技术验证场景`：显示技术验证用立方体和坐标轴。
  - `加载真实 OpenFOAM 3D Case`：加载真实 OpenFOAM 网格几何。
  - `加载压力云图`：读取真实字段 `p` 并渲染压力云图。

  - 日志区追加 `压力云图已加载：pRange=..., times=...`。

本阶段完成设计：
- `OpenFoamVtkCaseInfo` 新增 `point_arrays` 和 `cell_arrays`，用于记录 VTK 读取到的字段列表。
- `OpenFoamVtkService.inspect()` 不再只返回 block 和 time 信息，同时返回点字段和单元字段。
- `OpenFoamVtkService.build_geometry_filter()` 在构建几何前主动启用全部 VTK 字段数组。
- UI 层只负责按钮、错误提示和 VTK actor 组装。
- OpenFOAM case 读取逻辑继续留在 `OpenFoamVtkService`，避免 UI 直接操作 reader 细节。
- 本阶段为后续速度矢量、切面、流线和时间步选择预留了字段发现能力。

本阶段数据流设计：
- 用户点击 `加载压力云图` -> `MainWindow._load_pressure_cloud()` -> `OpenFoamVtkService.inspect()` -> 创建或复用 `case/foamdesk.foam` -> `vtkOpenFOAMReader` 读取 case 元信息 -> 返回时间步、block 数、点字段、单元字段。
- 用户点击 `加载压力云图` -> `OpenFoamVtkService.build_geometry_filter()` -> `vtkOpenFOAMReader` 读取真实 case -> `vtkCompositeDataGeometryFilter` 转为可渲染几何 -> `vtkPolyDataMapper` 选择 `p` 字段 -> `vtkActor` 渲染到内嵌 VTK 窗口。
- 如果 `p` 存在于点字段 -> 使用 `SetScalarModeToUsePointFieldData()`。
- 如果 `p` 只存在于单元字段 -> 使用 `SetScalarModeToUseCellFieldData()`。
- 如果压力范围最小值等于最大值 -> 临时扩展显示范围，避免颜色标尺退化。
- 本阶段不写 JSON，不改项目配置文件；只可能在 case 目录创建 `foamdesk.foam` 标记文件。
